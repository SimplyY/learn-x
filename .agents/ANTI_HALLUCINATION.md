# 防幻觉清单

> 本文件记录已发生的幻觉事故及其抽象规则。Agent 每次启动时必须阅读本文件。

## 高危·已发生 3 次：飞书文档内容变路径字符串

**现象**：用户要求把飞书记录/妙记内容生成飞书文档时，Agent 未实际获取到源内容，却声称创建成功。回读发现文档里只有一个 URL 路径字符串，没有任何实质内容。

**根因**：
1. 源内容获取失败（API 不支持、无权限、token 格式不匹配）时，Agent 没有如实报告失败
2. Agent 用源 URL 本身作为"内容"填入文档，制造创建成功的假象
3. 创建后没有回读验证文档实际内容

**规则**：
- 源内容获取失败时，必须如实报告"无法获取源内容"，不得用 URL、路径或任何占位符替代
- 创建飞书文档后，必须用 `lark-cli docs +fetch` 回读，确认文档包含实质内容（而非 URL 字符串）
- 未通过回读验证不得报告完成

**涉及产品**：
- 飞书「记录」(`/record/` URL)：新格式 token（如 `CjERrUOVRea7CwcF1N1cMexwnbe`），与妙记 API token 格式（`obcn...`）不兼容，当前 lark-cli 无法直接读取
- 飞书「妙记」(`/minutes/` URL)：token 格式 `obcn...`，可用 `lark-cli minutes +detail --transcript` 读取

## 通用防幻觉原则

1. **命令先行**：通过实际命令读取数据，不凭记忆或推理替代
2. **失败即报**：命令失败时如实报告，不虚构成功结果
3. **回读验收**：创建或更新外部对象后，必须回读并核对关键内容
4. **不编造内容**：没有读到的内容不得出现在输出中
