# 长文深度研究

对一篇长文（播客文字稿、文章、论文、访谈）做系统性深度研究，产出飞书文档，入库智慧之门 Base「研究」表。

## 核心理念

- **质量驱动字数** — 字数 = 10,000 × 信息量系数（上限 1.0）。质量评分（A/B/C）只用于研究后自评，不参与字数计算。不是硬凑，是质量的函数。
- **深度优先于广度** — 不追求面面俱到，在最关键的洞察上钻到底。
- **芒格之魂贯穿** — 六层思考框架不是附加步骤，而是每一阶段的思维底色。
- **动态 Skill 选择** — 必选 Skill 保证研究骨架，按文章类型 + 信息量系数动态确定可选 Skill。ljg-plain 按需选用，适合概念密集的文章。
- **可读性是硬约束** — 每个 Phase 产出必须用 Markdown 结构化（标题、列表、表格、引用块、加粗）。飞书文档不是纯文本，不用 Markdown 格式就是浪费。dense prose block 直接判不合格。
- **视角多样性** — 圆桌讨论必须包含与文章主题相关的不同文化视角。哲学/审美/自由主题强制加入东方视角（庄子、慧能、苏轼等）。

## 前置评估（研究开始前必须先输出）

在读完原文、进入 Phase 0 之前，先输出以下评估，作为全文的字数基准：

```
【前置评估】
- 原文约字数：X 字
- 信息量系数：Y（0.5-1.0，1.0=极高密度值得万字展开，0.5=可大幅压缩）
  - 判断依据：（一句话说明：概念密度、逻辑链复杂度、原创性、跨领域连接度）
- 预估质量目标：A / B（A=有原创洞见，B=分析到位）
- 目标字数：10,000 × 信息量系数 = Z 字（上限 10,000）
- 可选 Skill 决策：ljg-plain（是/否）、ljg-qa（是/否）
```

**信息量系数判断标准**：

| 系数 | 文章特征 | 例子 | ljg-plain 要求 |
|------|---------|------|---------------|
| 1.0 | 概念密集、逻辑链多层、每个段落都有不可删的实质内容 | 学术论文、深度分析报告 | 可选 |
| 0.9 | 高概念密度，有论证有案例，跨领域连接丰富 | 优质公众号深度长文、哲学思辨文章 | 可选 |
| 0.8 | 正常论述密度，有论证有案例，核心概念 3+ 个 | 优质公众号长文、播客文字稿 | 可选 |
| 0.7 | 有实质内容但夹杂叙事/案例/铺陈 | 叙事型长文 | 可选 |
| 0.6 | 核心观点比较集中，案例较多 | 访谈、经验分享 | 可选 |
| 0.5 | 核心观点可以用更少字说清楚，信息密度低 | 观点稀疏的长文 | 不需要 |

**可选 Skill 决策矩阵**：

| 信息量系数 | 文章概念密度 | ljg-plain | ljg-qa | 说明 |
|-----------|-------------|-----------|--------|------|
| 0.9-1.0 | 高（5+ 核心概念） | 可选 | 可选 | 概念多且深，白话转译和问答骨架都能显著提升深度 |
| 0.8-0.9 | 中高（3-5 个概念） | 可选 | 可选 | 概念需要白话讲透，问答按论证强度决定 |
| 0.7-0.8 | 中（2-3 个概念） | 可选 | 可选 | 两者都加可以显著提升深度 |
| 0.5-0.7 | 低（1-2 个概念） | 可选 | 可选 | 选一个匹配度高的即可 |

## 个人视角（前置评估后、Phase 0 前必须输出）

读完原文、输出前置评估后，必须读取 `01_core/道/` 目录下的全部文件（00_README.md、01_自我认知.md、02_人生导师.md、03_世界之道.md），以用户的认知框架为透镜，审视这篇文章对用户个人的价值。输出不超过 500 字。

**输出格式**：

```
## 个人视角

### 这篇文章对我有什么用？
（1-2 句，从用户的三重世界、当前重点、生命秩序出发，说清楚这篇文章与用户的关系）

### 核心指导
（3-5 条，每条一句。从用户的道/法/术框架出发，这篇文章提供了什么可行动的指导）

### 建议
（1-2 条，基于用户的张力与注意点，读完这篇文章应该注意什么、避免什么）

### 最值得深度看的章节
（按优先级排序，3-5 个章节，每个一句说明为什么值得看）
```

**思考框架（Agent 内化，不输出）**：

读道文件时抓住以下关键点，作为审视透镜：

- **生命秩序**：健康 > 生活 > 创造/工作 > 投资 > 系统。这篇文章触及哪一层？是否会反噬更底层？
- **三重世界**：原子（身体/关系）、比特（信息/连接）、向量（AI/模型）。文章在哪个世界？对另外两个世界有影响吗？
- **当前重点**：身体恢复、教育+AI、价值投资、Learn-X 系统、内在秩序重建。文章是否直接关联当前重点？
- **内在积分牌**：苏轼（着力即差）、孟岩（够与多、允许万物穿过自己）、孔子（因材施教、君子不器）。文章的观点是否与这些校准器一致或冲突？
- **剑与茧**：文章是在增强判断力，还是在制造信息茧房？是帮助用户守住主体性，还是诱导外包判断？
- **够与多**：这篇文章的信息是「够了」还是「多了」？读完是减少焦虑还是增加焦虑？
- **真实反馈**：文章是否连接到真实世界？是否可验证、可行动？

**硬约束**：
- 不超过 500 字
- 必须引用道文件中的具体概念（不能泛泛而谈「对投资有帮助」）
- 必须诚实——如果文章与用户当前重点关联弱，就说弱，不要强行关联
- 最值得看的章节必须具体到文章的实际段落/论点，不能是「全文都值得看」

## 质量评分机制

研究完成后，对产出做自评（A/B/C）：

| 维度 | A 级 | B 级 | C 级 |
|------|------|------|------|
| 洞察深度 | 有「原来如此」的认知跃迁 | 分析到位但不出意外 | 换个说法重复原文 |
| 原创性 | 产出原文没有的视角和框架 | 整合得当但无新框架 | 原文摘要换皮 |
| 连贯性 | 各阶段相互印证、层层递进 | 阶段间有联系但松散 | 各阶段各自为政 |
| 信息密度 | 每段都有不可删的实质内容 | 少量「正确的废话」 | 大段可删而不影响论证 |
| 可读性 | 标题/列表/表格/引用块运用得当，手机可流畅阅读 | 有结构但部分段落过密 | 大段纯文本，无结构 |
| 视角多样性 | 包含至少两种不同文化/学科视角 | 单一视角但分析深入 | 单一视角且浅 |

**自评 B 级或以下 → 回到薄弱环节重做，或用可选 Skill 补强。C 级直接废弃，不入库。**

**字数与质量的关系**：A 级产出字数通常接近目标字数（信息密度高，自然展开）；B 级可略少；C 级再多字也是废的。不要为凑字数而灌水，也不要为精简而砍洞察。

**字数差距预警**：如果实际字数 < 目标字数的 60%，说明研究深度不足或 Phase 覆盖不够。必须触发动态补偿（见下方「动态适配机制」）。优先检查：芒格之魂六层是否充分展开。ljg-plain 如已启用，检查转译质量。

## 各 Phase 字数参考

每个 Phase 没有硬性字数下限，但以下是保证深度所需的大致范围。Agent 在每个 Phase 结束后检查：如果产出远低于参考范围，说明该 Phase 可能没有充分展开。

| Phase | 内容 | 参考字数 | 说明 |
|-------|------|---------|------|
| 前置评估 + 个人视角 | 评估 + 道文件透镜 | 300-500 | 固定，不随目标字数变化 |
| Phase 0 | 全局地图（ljg-read） | 300-600 | 结构地图 + 核心概念 |
| Phase 1 | 本质深钻（ljg-think） | 600-1,200 | 两个洞察 × 四层钻探，每层 100-150 字 |
| Phase 2 | 概念解剖（ljg-learn） | 500-1,000 | 两个概念 × 八维解剖，每刀 50-80 字 |
| Phase 2.5a | ljg-plain（可选） | 500-1,000 | 3-5 个核心概念白话转译，每个 100-200 字 |
| Phase 2.5b | ljg-qa（可选） | 600-1,200 | 信息量系数 0.9+ → 4-6 问；0.7-0.8 → 3-4 问；0.5-0.6 → 2-3 问 |
| Phase 3 | 降秩分析（ljg-rank） | 500-1,000 | 七步工序 + 反生成验证 |
| Phase 4 | 约束分析（ljg-constraint） | 500-1,000 | 约束分层 + 解空间分析 |
| Phase 5 | 多视角圆桌（ljg-roundtable） | 800-1,500 | 3-5 人 × 2-3 轮交锋；哲学/审美主题加东方视角轮 |
| Phase 6 | 芒格之魂六层 | 1,000-1,800 | 每层 150-300 字，充分展开，不压缩 |
| Phase 7 | 盲区扫描（ljg-blind） | 500-1,000 | 五个盲区，每个必须包含「影响」展开（2-3 句） |
| 研究后自评 | 质量自评 | 100-200 | 固定 |

**累计参考范围**：约 5,600-11,500 字（不含可选 Skill 时约 4,500-8,500 字，含 ljg-plain + ljg-qa 时约 6,700-11,500 字（两者均为可选））。

**关键原则**：信息量系数 1.0 的文章，Agent 应尽量让每个 Phase 落在参考范围的中上段；系数 0.5 的文章可落在中下段。最终实际字数 = 各 Phase 自然产出之和，不灌水、不硬凑。**芒格之魂六层是字数增长的关键环节，不要在 Phase 6 压缩——这是全文整合框架，压缩了等于白做。**

## 动态适配机制

研究过程中，每完成一个 Phase 后检查累计字数 vs 目标字数。如果差距过大，触发补偿。

### 字数追踪节点

在以下节点检查累计字数（**含 Phase 2.5a/ljg-plain 和 Phase 2.5b/ljg-qa 如果启用**）：

| 节点 | 检查时机 | 预期累计 |
|------|---------|---------|
| 节点 1 | Phase 1 完成后 | 累计 ≥ 目标的 15% |
| 节点 2 | Phase 2 完成后 | 累计 ≥ 目标的 30% |
| 节点 3 | Phase 2.5（可选 Skill）完成后 | 累计 ≥ 目标的 45% |
| 节点 4 | Phase 4 完成后 | 累计 ≥ 目标的 60% |
| 节点 5 | Phase 5 完成后 | 累计 ≥ 目标的 75% |
| 节点 6 | Phase 6 完成后 | 累计 ≥ 目标的 90% |
| 节点 7 | Phase 7 完成后 | 累计 ≥ 目标的 95% |

### 补偿触发规则

如果当前节点累计字数 < 预期：

1. **轻度不足（差距 < 15%）**：下一个 Phase 尽量展开，不要压缩。
2. **中度不足（差距 15-30%）**：
   - 如果 ljg-plain 还没启用 → 建议启用
   - 如果 ljg-qa 还没启用 → 建议启用
   - 或选择一个已完成的 Phase 做更深展开
3. **严重不足（差距 > 30%）**：
   - 建议启用 ljg-plain + ljg-qa（如果还没用）
   - Phase 6 芒格之魂做完整六层展开（每层 200-300 字，不压缩任何一层）
   - Phase 5 圆桌讨论加一轮（如果只有 2 轮）

### Phase 补救流程

如果全部 Phase 完成后实际字数 < 目标的 60%，按以下顺序补强：

1. 检查 ljg-plain 是否适合启用 → 如果适合且未启用，建议启用
2. 检查芒格之魂六层是否每层都充分展开（≥ 150 字/层）→ 如果不够，展开
3. 检查 ljg-qa 是否适合启用 → 如果适合且未启用，建议启用；如已启用但问答不足，建议扩展
4. 检查圆桌是否有视角多样性（是否缺东方视角轮）→ 追加一轮
5. 如果所有 Phase 都达标但总字数不足 → 检查信息量系数是否设得太高，可能需要在自评中说明原因

## Skill 体系：必选 + 可选

### 必选 Skill（始终执行，保证研究骨架）

| 顺序 | Skill | 标题备注 | 作用 |
|------|-------|---------|------|
| 0 | ljg-read | Phase 0：全局地图 | 通读建图，产出结构地图 + 关键概念 |
| 1 | ljg-think | Phase 1：本质深钻 | 选 2 个洞察纵向深钻到底 |
| 2 | ljg-learn | Phase 2：概念解剖 | 选 2 个概念做八维解剖 |
| 3 | ljg-rank | Phase 3：降秩分析 | 找底层生成器——什么在撑着这个领域？砍到不可再少 |
| 4 | ljg-constraint | Phase 4：约束分析 | 找边界约束——什么框住了这个问题？硬/软/自设三层 |
| 5 | ljg-roundtable | Phase 5：多视角圆桌 | 3-5 位人物多轮交锋，哲学/审美主题强制加入东方视角 |
| 6 | 芒格之魂 | Phase 6：六层思考 | 六层框架整合全部发现，每层 150-300 字充分展开 |
| 7 | ljg-blind | Phase 7：盲区扫描 | 找出文章没说的、绕开的、默认的前提假设。任何研究不扫盲区就是伪闭环 |

### 可选 Skill（按需增补，增强研究深度）

| Skill | 标题备注 | 适合什么内容 | 产出 |
|-------|---------|-------------|------|
| ljg-plain | Phase 2.5a：白话转译 | 概念密集的文章（3+ 核心概念），技术性/哲学性强 | 把核心概念用大白话讲透，让非专业读者也能 grok |
| ljg-qa | Phase 2.5b：问答骨架 | 论证性强、逻辑链清晰的文章 | 抽出「为什么—怎么—边界」问答链，把作者推理骨架翻出来 |

**选择原则**：
- 读完文章后按决策矩阵（见前置评估）确定 ljg-plain 和 ljg-qa 是否启用
- ljg-plain 对概念密集的文章可以启用，能显著提升可读性
- ljg-qa 问答数量与信息量系数挂钩：系数 0.9+ → 4-6 问；0.7-0.8 → 3-4 问；0.5-0.6 → 2-3 问
- 可选 Skill 的产出必须融入整体研究，不能是独立附录

### 各 Phase 详细说明

#### Phase 0：全局地图（ljg-read）

一次性产出：一句话摘要、段落骨/肌/筋分类、ASCII 结构地图、3-5 个核心概念。

**输出格式要求**：
- 用 `## Phase 0：全局地图` 标题
- 一句话摘要用 `>` 引用块
- 段落分类用无序列表（`- 骨：...` / `- 肌：...` / `- 筋：...`）
- ASCII 结构地图用代码块包裹
- 核心概念用 `### 核心概念` 子标题 + 无序列表，概念名 **加粗**

#### Phase 1：本质深钻（ljg-think）

选 2 个最关键的洞察，纵向下坠：表象 → 机理 → 原理 → 公理。每层只回答「为什么是这样」，不横向扩散。钻到底：人性基本结构/物理定律/逻辑本身。

**输出格式要求（强制执行）**：
- 用 `## Phase 1：本质深钻` 标题
- 每个洞察用 `### 洞察一：{洞察名}` / `### 洞察二：{洞察名}` 子标题
- **钻探链必须用表格呈现**，每层一行：

```
| 层级 | 内容 | 判断 |
|------|------|------|
| 表象 | {现象描述} | 原文说... |
| 机理 | {运作机制} | 因为... |
| 原理 | {底层原理} | 本质上... |
| 公理 | {不可再分} | 到底了：... |
```

- 每层「到底了」的结论用 `>` 引用块单独标出
- 禁止纯文本段落堆砌——表格是强制格式，不用表格 = 不合格

#### Phase 2：概念解剖（ljg-learn）

选 2 个核心概念，八维解剖（历史/辩证/现象/语言/形式/存在/美感/元反思），每刀 2-3 句。压成一句顿悟 + 公式 + ASCII 图。

**输出格式要求**：
- 用 `## Phase 2：概念解剖` 标题
- 每个概念用 `### 概念：{概念名}` 子标题
- 八维解剖用无序列表，维度名 **加粗**（如 `- **历史**：...`）
- 顿悟句用 `>` 引用块
- 公式用代码块
- ASCII 图用代码块

#### Phase 2.5a：白话转译（ljg-plain）— 可选

把核心概念用大白话讲清楚，让没有专业背景的读者也能抓住要害。每个概念：白话解释 + 一句话总结 + 对用户的启发（关联道/法/术框架）。

**输出格式要求**：
- 用 `## Phase 2.5a：白话转译` 标题
- 每个概念用 `### {概念名}：「{一句话白话定义}」` 子标题
- 白话解释用正文段落（2-4 句，用生活化的比喻和例子）
- 一句话总结用 `**一句话**：...`
- 对用户的启发用 `**对我的启发**：...`（关联道/法/术/内在积分牌/当前重点）

**格式示例**：

```
### 物自体：「事物的本来面目，你永远看不到」

你面前有一张桌子。你以为你看到的是「桌子本身」，但实际上……

**一句话**：事物存在，但人只能看到被自己认知框架过滤过的版本。

**对我的启发**：在 AI 时代，你同样戴着一副「AI 框镜」——你看到的所有 AI 输出，都是被模型训练数据和算法过滤过的。
```

- 转译 3-5 个概念（系数 0.9+ → 5 个；0.8 → 3-4 个；0.7 → 2-3 个）
- 不要用哲学术语解释哲学术语——用日常生活的例子

#### Phase 2.5b：问答骨架（ljg-qa）— 可选

抽出「为什么—怎么—边界」问答链。每个 Q 切要害（不是「定义是什么」而是「为什么成立」「代价是什么」），每个 A 四段式（结论 + 形式化 + 论证步 + 边界）。Q 链有方向，读者顺着走能复现作者整套推理。

**问答数量与信息量系数挂钩**：
- 系数 0.9-1.0 → 4-6 个问答
- 系数 0.7-0.8 → 3-4 个问答
- 系数 0.5-0.6 → 2-3 个问答

**ljg-qa 输出格式**：
- 用 `## Phase 2.5b：问答骨架` 标题
- 每个 Q-A 对用 `### Q{n}：{问题}` 子标题
- A 的四段式用有序列表（`1. **结论**：...` / `2. 形式化：...` 等）
- 结论句 **加粗**

#### Phase 3：降秩分析（ljg-rank）— 必选

找出背后真正独立的几根生成器，砍到不可再少。七步工序（铺现象→列候选→递归下沉→合并同源→砍→反生成→预测+变更双测），过 Deutsch 两关：解释力（能推出清单外的现象）+ 难以变更（动一处就崩）。这是研究的「找底」环节——找到这个领域最底层的那几根线。

**输出格式要求**：
- 用 `## Phase 3：降秩分析` 标题
- 领域名称用 `### 领域：{领域名}` 子标题
- 基本假设用无序列表，每条 **加粗** 假设名
- Root rank 用表格（`| 生成器 | 硬度 | 说明 |`）
- 反生成验证用无序列表（`- {现象} ← {生成器} 解释：...`）
- ASCII 结构图用代码块（按 rank 形状选取景框）
- 关键结论用 `>` 引用块

#### Phase 4：约束分析（ljg-constraint）— 必选

找到框住问题的硬约束/软约束/自设约束，看清这组约束框出的解空间。七步工序（铺约束候选→分层归位→真伪定性→找约束错配→找矛盾对→找身份约束→两把透镜收口）。不要把约束写成普通困难——拿掉约束后允许的行为必须会变。这是研究的「找边」环节——找到这个问题的边界和形状。

降秩和约束是研究的两根支柱：**降秩往下挖，找到底；约束往边上摸，找到边。** 底和边都找到了，这个问题的解空间才真正浮现。

**输出格式要求**：
- 用 `## Phase 4：约束分析` 标题
- 题面补全用 `>` 引用块
- 约束组用表格（`| 约束 | 硬度 | 身份约束？ | 说明 |`）
- 解空间分析用无序列表
- 解释力验证（行为→约束）用表格（`| 观察到的行为 | 被哪条约束逼出来的 |`）
- ASCII 结构图用代码块（解释链图/硬度嵌套图/约束矛盾图）
- 关键结论用 `>` 引用块

#### Phase 5：多视角圆桌（ljg-roundtable）

提炼核心争议议题，邀请 3-5 位真实人物，2-3 轮交锋，每轮 ASCII 结构图，收场画知识网络图。Agent 独立模拟，不需要用户交互。

**视角多样性规则（强制）**：
- 哲学/审美/自由/认知主题 → 必须加入东方视角轮（庄子、慧能、苏轼等，至少 1 位）
- 投资/市场主题 → 必须加入实践派视角（芒格、李录等，至少 1 位）
- 科技/AI 主题 → 必须加入人文视角（哲学家、艺术家等，至少 1 位）
- 如果文章跨多个主题，优先选择最核心主题的视角多样性要求

**输出格式要求**：
- 用 `## Phase 5：多视角圆桌` 标题
- 核心议题用 `### 议题` 子标题
- 参与者用表格（`| 人物 | 核心立场 | 代表观点 |`）
- 每轮交锋用 `### 第{n}轮：{主题}` 子标题
- 每位发言用 `> **{人物名}**：...` 引用块
- ASCII 交锋结构图用代码块
- 知识网络图用代码块
- 收场总结用无序列表

#### Phase 6：六层思考（芒格之魂）

底层 → 第二层（领域同构）→ 第三层（反转假设）→ 第四层（变换尺度）→ 第五层（简化支点）→ 顶层（整合新洞见）。每层充分展开（150-300 字），不急于上升。**这是全文整合框架，不要在 Phase 6 压缩——压缩了等于白做。**

**输出格式要求**：
- 用 `## Phase 6：芒格之魂·六层思考` 标题
- 每层用 `### 第{n}层：{层名}` 子标题
- 每层核心发现用 `>` 引用块
- 每层展开 2-3 段（150-300 字），包含具体例子或论证，不能只有一句话
- 层间递进关系用一句 `**→ 上升逻辑**：...` 连接
- 顶层整合用表格（`| 维度 | 发现 |`）或对比列表
- 禁止纯文本段落——每层至少有一个引用块或列表

#### Phase 7：盲区扫描（ljg-blind）— 必选

找出文章没说的、绕开的、默认的前提假设。五种盲区信号：绕开点、空转框架、单一取景框、未检前提、相邻空缺。

**每个盲区必须包含**：
1. 盲区描述（文章绕开了什么）
2. 具体例证（用 `>` 引用块标注「原文绕开了...」）
3. **影响**（2-3 句展开：如果不扫这个盲区会怎样？对理解/行动有什么后果？）

**重点盲区展开要求**：
- 「单一取景框」盲区（如只有西方哲学视角）→ 必须展开，说明缺失了什么视角
- 「未检前提」盲区（如默认某个能力可训练）→ 必须展开，说明如果前提不成立会怎样

**输出格式要求**：
- 用 `## Phase 7：盲区扫描` 标题
- 每种盲区用 `### 盲区{n}：{类型} — {盲区名}` 子标题
- 盲区描述用正文段落
- 具体例证用 `>` 引用块
- 影响分析用 `**影响**：...`（2-3 句）
- 整体盲区总结用表格（`| 盲区 | 类型 | 严重程度 | 影响 |`）

### Phase 8：排版后处理

Phase 产出内容后，执行一次排版优化 Pass，不更改内容，只匹配飞书文档排版能力。

#### 排版 Pass 规则

1. 读取完整文档，验证每个 Phase 的格式要求是否满足
2. 标题层级保持：`#` 文档标题 → `##` Phase → `###` 子节
3. 段落不超过 200 字，超长段落自动拆分
4. 关键词统一 **加粗**
5. ASCII 图用代码块包裹
6. **可读性检查**：扫一遍全文，任何超过 5 行连续纯文本（无标题/列表/表格/引用块/加粗/代码块）的段落，必须拆分或加结构

## 触发流程（飞书入口）

1. 用户发长文本或文件，@codex 小助手 说"深度研究这篇"
2. Agent 读取全文（文件用 `cat`，URL 用 `read` skill 或 `wechat-article-to-markdown`，粘贴文本直接用）
3. **读取道文件**：`cat 01_core/道/00_README.md 01_core/道/01_自我认知.md 01_core/道/02_人生导师.md 01_core/道/03_世界之道.md`
4. **输出前置评估**：原文字数、信息量系数、目标字数、**可选 Skill 决策（ljg-plain 可选、ljg-qa 可选）**
5. **输出个人视角**：基于道文件，审视文章对用户的价值（500 字以内）
6. 按必选（ljg-read → ljg-think → ljg-learn → [ljg-plain 如果启用] → [ljg-qa 如果启用] → ljg-rank → ljg-constraint → ljg-roundtable → 芒格之魂 → ljg-blind）顺序执行研究
7. 每个 Phase 结束后检查字数追踪节点，触发补偿（如需要）
8. 全部完成后自评质量（A/B/C），B 级以下用可选 Skill 补强或重做薄弱环节
9. 排版后处理，组装 Markdown，入库
10. 回复用户：研究主题 + 质量评级 + 实际字数 vs 目标字数 + 一句话洞察 + 飞书文档链接（文档开头已标注质量/字数）

## 重要约束

- **前置评估** — 研究开始前必须先输出字数基准、信息量系数、目标字数、**Skill 决策**。这是全文的锚。
- **ljg-plain 可选** — 概念密集的文章可以启用 ljg-plain。白话转译能显著提升可读性，让非专业读者也能抓住核心概念。
- **动态 Skill 选择** — 按决策矩阵（信息量系数 + 文章概念密度）确定可选 Skill。不要机械套用所有 Skill，也不要死守默认流程。
- **质量自评** — 研究完成后必须自评 A/B/C，包含视角多样性维度。B 级以下不能入库，先补强。
- **字数上限** — 硬上限 10,000 字。目标字数 = 10,000 × 信息量系数（上限 1.0）。不硬凑，但实际字数不应低于目标的 60%。不足时触发动态补偿。
- **动态适配** — 7 个节点追踪（含 Phase 2.5 和 Phase 5）。差距 > 15% 触发补偿。全部完成后 < 目标 60% 必须补强，优先检查芒格之魂六层展开。ljg-plain 如已启用，检查转译质量。
- **可读性是硬约束** — 每个 Phase 必须用 Markdown 结构化（标题、列表、表格、引用块、加粗）。dense prose block 直接不合格。嵌在 Phase 说明里的格式要求是强制执行的，不是建议。
- **芒格之魂不是最后才用** — 贯穿前五阶段的思维底色，Phase 6 是显式整合。每层 150-300 字充分展开，不压缩。
- **视角多样性** — 圆桌讨论必须包含与文章主题相关的不同文化/学科视角。哲学/审美/自由主题强制加入东方视角。
- **ljg skill 是思维框架** — Agent 内化每个 skill 的思考方式，不是机械套用格式。但输出格式必须遵守本 SKILL 的约束。
- **对抗性审查** — 研究完成后必须逐项过自审清单。最核心的洞察是否经得起反转？是否有伪深度？每个 Phase 的输出能独立阅读吗？
- **排版约束** — 段落不超过 200 字；使用 #/##/### 标题层级；关键词用 **加粗**；要点用 - 列表；关键结论用 > 引用；ASCII 图用代码块包裹。
- **手机场景** — 飞书群里只回复结论和链接，完整研究在文档里。

## 对抗性自审清单

研究完成后，逐项检查（每项必须通过才能入库）：

1. **反转测试** — 最核心的洞察反过来还成立吗？如果反过来也对，说明洞察太模糊，必须重做 Phase 1
2. **伪深度检测** — 六层思考中，哪几层是真正的认知跃迁，哪几层只是换个说法重复？如果 ≥ 3 层是重复，Phase 6 不合格
3. **信息密度** — 有没有整段都是「正确的废话」？砍掉后论证是否依然完整？如果砍掉 ≥ 20% 内容不影响论证，说明有灌水
4. **来源忠实度** — 引用是否准确？有没有为了论证方便而扭曲原意？抽查 3 处引用对照原文
5. **可读性审查** — 每个 Phase 的输出能独立阅读吗？手机屏幕上看是否清晰？有没有超过 5 行的纯文本墙？
6. **格式审查** — ljg-think 用了表格吗？ljg-rank 有结构图吗？约束分析有约束表格吗？盲区有类型分类 + 影响展开吗？
7. **视角多样性审查** — 圆桌讨论是否包含至少两种不同文化/学科视角？哲学/审美主题是否加入了东方视角？如果没有，必须补一轮
8. **补偿机制检查** — 实际字数是否 ≥ 目标字数的 60%？如果不足，是否已触发补偿？补偿后是否达标？
9. **ljg-plain 检查** — 概念密集的文章，是否启用了 ljg-plain？如果没启用且概念确实难懂，建议补上
10. **自主性声明** — 所有阶段（包括圆桌讨论）均为 Agent 自主模拟，不需要用户交互。文档末尾必须标注研究完成日期。
