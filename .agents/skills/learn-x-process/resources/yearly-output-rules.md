# Weekly Output 输出要求

本文件维护 `learn-x-process` 的核心输出要求。

`SKILL.md` 说明这个 Skill 是什么、什么时候用、边界在哪里；本文件说明最终写给人的 Weekly Output 应该长什么样。

---

## 核心定位

Weekly Output 不是自动分类结果、脚本日志、材料摘要，也不是候选清单堆叠。

它是给人认真阅读的**每周认知审稿台**，目标不是只做判断，而是帮助用户完成六件事：

1. **摘要**：看见本周真正重要的输入、行动和变化。
2. **原料回顾**：保留本周高质量输入源的核心要点，避免好材料被综合加工后丢失。
3. **洞察**：从材料中提炼出对自我、项目、学习、AI、教育、投资和生活有价值的认知增量。
4. **盲点**：发现用户可能忽略、低估、高估或尚未验证的部分。
5. **长期视角**：把本周问题放到更长时间、更大系统、更真实生活中重新理解。
6. **行动闭环**：检查认知是否进入行动，行动是否获得反馈，反馈是否修正判断。

Weekly Output 的价值不在于“记录完整”，而在于：

> 把输入变成理解，把理解变成判断，把判断变成行动，把行动变成反馈。

但要特别注意：

> 高质量原材料不能被 AI 的综合摘要压扁。
> 对 Flomo、AI 聊天、阅读笔记、关键行动记录等高价值输入源，应保留其最核心的原始思想、问题意识和表达锋芒。

它应该回答：

* 本周真正发生了什么？
* 本周最重要的核心摘要是什么？
* 哪些输入源本身质量很高，值得单独回顾？
* 这些高质量输入源分别贡献了什么核心思想？
* 哪些洞察值得保留？
* 我可能没有看见什么？
* 从 3 个月、1 年、3 年后看，这件事还重要吗？
* 哪些判断已经进入行动？哪些还停留在想法层？
* 下周最小推进动作是什么？

---

## 字数原则

Weekly Output 不强制固定 1000 字以内。

默认控制：

* 常规周：1000-1500 字；
* 材料密度高、真实行动多、高质量输入源多、洞察强时：可扩展到 1800 字左右；
* 输入质量不足时：可以少写；
* 不为了字数凑模块；
* 不为了简短删除关键洞察；
* 不为了综合而抹平高质量原材料。

核心原则：

> 密度优先，重要性优先，可读性优先，关键原料优先。

---

## 输入来源平等，但证据强度不同

不要预设主干一定来自 `03_input/weekly/YYYY-Www/01_inbox/ai.md`。

本周主线可能来自：

* `00_log/`：自我反馈，我如何理解自己的状态和变化。
* `01_inbox/`：信息输入，我接收了什么。
* `02_action/`：行动证据，我做了什么，世界如何反馈。

判断主线时看内容质量、重复频率、现实重要性、洞察价值和行动价值，而不是看来源目录。

证据优先级：

> 行动证据 > 自我反馈 > 反复出现的高价值输入 > 一次性信息查询 > 工具噪声。

但来源目录不决定价值。最终看材料是否改变了理解、判断、行动或长期系统。

必须明确区分：

* `00_log/` 是自我反馈输入，不是 Memory。
* Memory 是人工确认后的输出，不和 `00_log/` 混淆。
* AI 对话不是天然主干；只有当它改变判断、行动或长期问题意识时，才成为主干。
* Flomo、AI 聊天、阅读笔记等高质量输入源，即使没有直接行动，也可能值得保留核心思想，因为它们可能是长期判断和写作素材的源头。

## AI 智能责任

脚本只负责确定性工作：

* 读取文件
* 解析格式
* 清洗文本
* 合并与去重
* 编号
* 统计来源覆盖
* 生成 Input Pack 和 Process Pack

最终 Weekly Output 必须由 AI 按本文件进行智能判断和改写。

如果脚本生成的候选不适合人读，AI 要主动压缩、重排、改写、删除，而不是照搬。

不要把脚本关键词命中的结果当作最终判断。

AI 的加工应分三层：

1. **保留原料核心**：先看哪些输入源本身值得回顾，提取其最有价值的核心要点。
2. **提炼洞察判断**：再把不同来源合并，提炼模式、洞察、盲点和长期判断。
3. **转化行动闭环**：最后判断哪些内容需要进入下周行动、复盘或人工确认。

不要一上来就把所有材料融成统一判断，导致高质量原材料的细节、锋芒和独特问题意识丢失。

---

## Process Pack 不是报告

`04_output/_dist/yearly/YYYY/process-pack.md` 只用于让 AI 稳定读取材料。

它可以包含：

* 本周来源覆盖
* 每条材料的 source id
* 原始路径
* 文件时间
* 清洗后的正文片段
* 重复信息

它不应该包含：

* 道 / 法 / 术判断
* Prompt / Skill 判断
* 入库建议
* 行动优先级
* 像最终周报一样的结论段落

Weekly Output 必须基于 Process Pack 再进行摘要、重要输入源回顾、洞察、盲点识别、长期视角判断和行动闭环复盘。

---

## 输出质量标准

Weekly Output 必须满足：

* **可靠**：不编造输入中没有的信息；不确定就标注观察或待确认。
* **重要**：优先放影响理解、判断、行动、长期资产的内容。
* **可读**：像给人看的认知审稿台，不像机器分桶或脚本日志。对于复杂、多维信息，尤其是摘要、本周总览这种，一定要分点分段回答，不要出现太多一段超过100字甚至200字的内容。。
* **有摘要**：能让用户快速看见本周真正重要的内容。
* **有原料回顾**：能保留高质量输入源的核心要点，而不是全部融成 AI 的二次判断。
* **有洞察**：不能只总结发生了什么，还要提炼“这说明什么”。
* **能发现盲点**：指出高估、低估、未验证、过度系统化或行动不足之处。
* **有长期视角**：必要时从更大时间、空间和系统维度看问题。
* **可操作**：下周行动必须具体、有完成边界。
* **可追溯**：重要判断保留最小来源路径（所有来源都要用 markdown 引用块包裹，并只在最后汇总输出）。
* **宁少勿滥**：每类候选可以是 0 条、1 条、2 条，不硬凑。
* **保守沉淀**：未验证的漂亮句子不能直接进入道 / 法 / 术。

# 推荐输出结构

## 0. 本周总览

写上精准生成时间。

用 100-150 字说明本周最核心发生了什么，不列主题词，不写流水账。

必须回答：

* 本周真正的主线是什么？
* 哪个问题反复出现？
* 有没有明显变化、转向或新的张力？
* 行动闭环大致处于什么状态？
* 哪些来源贡献了最关键证据？
* 是否存在高质量输入源值得单独回顾？

---

## 1. 核心摘要

用 3-5 条 bullet 总结本周最重要内容。

要求：

* 不是流水账；
* 不是来源清单；
* 每条尽量写成“发生了什么 + 为什么重要”；
* 每条尽量不超过 60 字；
* 优先保留影响长期判断、行动、生活状态和项目推进的内容。

示例格式：

* Learn-X 从“记录系统”进一步转向“认知审稿台”，重点变成压缩、确认和行动闭环。
* AI 赋能真实场景开始浮现，但多数判断仍缺现场观察和外部反馈。

---

## 2. 重要输入源回顾

本节用于保留本周高质量原材料的核心要点，防止它们被综合摘要和洞察加工后丢失。

不是所有来源都要进入本节。只选择本周最有价值的 2-5 个输入源或输入源组。

优先考虑：

* 高质量 Flomo 笔记；
* 高质量 AI 聊天；
* 高质量读书笔记；
* 关键会议 / 关键微信聊天 / 关键行动记录；
* 对用户长期判断、写作、项目、人生问题有明显价值的原始材料。

不要做“来源核对表”。
不要把所有文件逐条列出。
不要只写路径。
要写清楚这个输入源本身贡献了什么核心内容。

每个输入源格式：

### 输入源标题

* 核心要点：
* 最有价值之处：

要求：

* 每个输入源控制在 80-120 字；
* 可以保留 1 句极短原文或关键表达，但不要大段摘录；
* 重点保留原材料本身的思想密度、问题意识和独特表达；
* 后续洞察可以在此基础上再加工，但不能替代本节。

示例：

### Flomo：自由、流动与内在计分牌

* 核心要点：本周 Flomo 反复出现水、流动、自由、内在计分牌等表达，指向用户对主体性、生活节奏和精神秩序的自我校准。
* 最有价值之处：它不是项目任务，而是长期人生系统的底层情绪和价值信号。

---

## 3. 本周关键洞察

保留 2-4 条。

洞察不是普通总结。只有满足以下至少一项，才进入本节：

* 改变了用户对自己、项目、AI、教育、投资、生活的理解；
* 揭示了一个反复出现的模式；
* 能迁移到未来多个场景；
* 能改变下周行动方式；
* 能解释过去一段时间的反复纠结。

每条格式：

### 洞察标题

* 洞察：
* 为什么重要：
* 可能影响：

必要时加入长期视角：

* 放到 3 个月、1 年、3 年后看，它是否仍重要？
* 它只影响工具效率，还是影响生活方式、关系、事业、资产或健康？
* 它属于输入系统、行动系统、反馈系统、资产系统、教育创业系统，还是人生秩序？

不要为了显得深刻写空泛哲学。长期视角必须服务本周材料。

---

## 4. 盲点与反向提醒

保留 1-3 条。

重点发现：

* 用户可能高估了什么；
* 用户可能低估了什么；
* 哪些判断还缺现实证据；
* 哪些系统设计可能过度复杂；
* 哪些内容只是 AI 对话中的漂亮表达；
* 哪些行动还停留在想法层；
* 哪些长期目标可能正在被短期工具问题稀释；
* 哪些东西看似重要，其实只是短期情绪或工具噪声；
* 哪些高质量输入源虽然很有洞察，但尚未被真实选择验证。

输出风格要直接，但不要夸张。盲点必须服务行动和长期校准，不做泛泛批评。

格式：

### 盲点标题

* 可能的问题：
* 为什么需要警惕：
* 建议校准：

---

## 5. 做中学复盘

这是 Weekly Output 的核心部分之一，并承担行动闭环检查。

重点不是新增更多行动，而是评估本周已经发生的真实行动质量。

### 评分

先给本周整体「做中学评分」：

* C：几乎只有输入、聊天、思考，没有真实行动。
* B：有行动或产物，但反馈弱，更多停留在想法层。
* A：有明确行动，也获得反馈，并修正了部分判断。
* S：行动 → 反馈 → 修正 → 再行动，形成完整学习闭环。

评分必须严格：

* 没有第二轮行动或外部/现实反馈，原则上不能评 S。
* 不要把“完成产物”直接等同于“获得反馈”。
* 需要标注反馈类型：内部反馈 / 外部反馈 / 暂无反馈。

### 输出格式

#### 主题标题，20 字以内

* 行动：
* 反馈：
* 反馈类型：
* 学到什么：
* 未闭环：
* 当前评分：
* 下一步最小动作：

判断原则：

* 优先看已经发生的行动，而不是制造新计划。
* 如果本周已经做了核心事情，重点是如何提高质量。
* 如果行动不足，再提出一个最小补充动作。
* 不写庞大计划。
* 行动闭环检查并入本节，不单独开固定模块。

---

## 6. 下周最该做的 3 件事

只给 3 件，使用 Markdown checkbox。

下周任务必须来自本周核心摘要、重要输入源回顾、关键洞察、盲点、行动断点或未闭环判断，不能凭空生成宏大计划。

每件事必须满足：

* 一周内可完成；
* 有明确产物或反馈；
* 能推动一个核心问题闭环。

格式：

* [ ] 任务：

  * 对应问题：
  * 最小完成标准：

---

## 7. 道 / 法 / 术候选

这里只输出候选，不代表已入库，必须等待人工确认。

道要极其保守。大多数周报都不应该新增「道」。

* 道：长期价值判断、人生方向、主体性原则。
* 法：可迁移的判断框架。
* 术：可执行方法、流程、模板、复盘动作。

输出规则：

* 道：默认 0 条；除非本周出现长期稳定、可反复指导选择的新证据。
* 法：最多 2 条；必须可迁移到其他场景。
* 术：最多 3 条；必须能被下周行动验证。
* 每条不超过 50 字。
* 长期反复出现的判断，如果本周没有新证据，不重复作为「道」输出，只写“延续观察”。
* 有些内容适合保留为“重要输入源回顾”或“洞察”，不一定要升级为道 / 法 / 术。

格式：

* [ ] 道：
* [ ] 法：
* [ ] 术：

如果没有强信号，可以写：

> 本周不建议沉淀新的道 / 法 / 术。

---

## 8. 证据缺口 / 输入缺口

本节不是固定模块。

只有当缺口会影响判断、行动或沉淀时才输出，最多 3 条。

每条必须能转化为：

* 下周行动；
* 降权判断；
* 暂缓判断；
* 补充输入要求。

不要为了完整而固定输出缺口。

---

# 必要时才提

以下能力不作为 Weekly Output 固定模块：

* 图谱候选
* 第一性原理候选
* 写作 / Demo 候选
* Prompt 候选
* Skill 候选
* 来源覆盖
* 删除 / 降权清单

只有当本周材料中出现强信号，且它们能直接服务重要输入源回顾、洞察、盲点、长期视角或行动闭环时，才可以用 1 条简短说明带过。

不要为它们单独开固定 section。

---

# 不要这样输出

* 不要把 50 条候选逐条铺开。
* 不要机械重复“原始来源 / 核心表达 / 建议层级”。
* 不要把脚本关键词命中的结果当作最终判断。
* 不要把 AI 对话天然视为主干。
* 不要固定输出「观察、降权或删除」。
* 不要固定输出「来源覆盖」。
* 不要为了完整而牺牲可读性。
* 不要只做判断而没有摘要、洞察和盲点。
* 不要只做摘要而没有判断和行动闭环。
* 不要把高质量输入源全部融成二次判断，导致原材料核心要点丢失。
* 不要把“重要输入源回顾”写成文件路径清单。
* 不要把低质量来源硬塞进“重要输入源回顾”。
* 不要把未验证的漂亮句子直接写成「道」。
* 不要把旧判断重复包装成新沉淀。
* 不要把内部产物误判成外部反馈。
* 不要把做中学复盘写成大论文、课程设计或复杂项目计划。
* 不要为了长期视角写空泛哲学。
* 不要固定输出图谱、第一性原理、Prompt、Skill、写作或 Demo 候选。

---

# 输出后自检

生成 Weekly Output 后，必须内部自检并据此修订，不需要输出完整自检过程。

自检问题：

1. 有没有只做判断，缺少核心摘要？
2. 有没有只做摘要，缺少洞察？
3. 有没有识别本周高质量输入源？
4. 有没有保留高质量输入源的核心要点，而不是全部融成 AI 的二次判断？
5. 有没有发现至少 1 个有价值的盲点或反向提醒？
6. 有没有把本周问题放到更长期或更大系统中看？
7. 有没有把输入总结误写成行动？
8. 有没有把内部产物误判成外部反馈？


11. 下周 3 件事是否都有最小完成标准？
12. 是否有可以删除的空话、漂亮话、重复话？
13. 字数是否符合材料密度，而不是机械压缩或机械扩写？
14. 是否保留了足够但不过量的来源路径？

---

# 最小判断

如果本周输入质量不足，输出也要诚实：

* 可以少写。
* 可以说“本周不适合沉淀道”。
* 可以只给核心摘要和盲点。
* 可以说“本周没有值得单独回顾的高质量输入源”。
* 可以建议补充更好的行动证据、自我反馈输入或原始笔记。

Weekly Output 的价值不在长度，而在帮助用户形成更真实的理解、更少的盲点、更稳的长期判断、更好地保留高质量原材料，并推动更小但有效的行动。
