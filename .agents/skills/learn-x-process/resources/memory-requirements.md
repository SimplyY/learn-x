# Weekly Memory 输出要求

Weekly Memory 是人工审核后的极薄跨周上下文，不是正式长期资产，也不是 Weekly Output 摘要。

## 定位

它保存本周已经人工确认、值得下周继续复用或追踪的少量判断。

用途：

- 给后续 Chat Pack / AI 对话提供更好的上下文。
- 快速回看本周真正确认的重要判断。
- 支持跨周追踪反复出现的主题。
- 作为月度沉积到正式资产的干净输入。

## 输入

主要读取：

- `04_output/weekly/YYYY-WW.md`
- `04_output/_dist/YYYY-WW/memory-candidates.md`

重点看：

- 已勾选 checkbox：`- [x]`
- 明确标记为「保留」「进入记忆」「继续追踪」「重要」「确认」的内容
- 少量明显属于本周核心判断、核心行动、核心主题的内容

## 输出

写入季度文件：

`01_core/memory/YYYY-QN.memory.md`

每周新增或更新一个小节：

```md
## YYYY-WW
```

## 质量标准

- 每周 4-5 条以内。
- 每条 20-30 字左右，最多不超过 40 字。
- 每周最好 150 字以内，最多不超过 250 字。
- 不追求完整。
- 不保存过程。
- 不保存所有候选。
- 不机械复制 Weekly Output。
- 必须压缩、合并、去重、提炼。
- 不把未经人工确认的漂亮表达写成长期信念。
- 不替代 `道/`、`法/`、`术/` 正式资产。

## 推荐结构

```md
# Learn-X Memory｜YYYY-QN

## YYYY-WW

- ...
- ...
- ...

### 继续追踪

- ...
```

如果本周确认内容很少，可以只写 1-3 条 bullet。不要为了季度文件完整而填充。

## 压缩口径

- 写成索引句，不写解释句。
- 优先保留判断名、行动名和追踪主题。
- 去掉原因、过程、背景和修饰语。
- 复杂判断拆成短条，不写长句。
