# Weekly Output 核心输出要求

本文件定义 `learn-x-process` 生成 Weekly Output 时的判断原则、输出结构和质量标准。

## 1. 核心定位

Weekly Output 是 Learn-X 个人认知进化系统中的每周认知审稿环节。

它不是普通周报、材料摘要、脚本日志、候选清单或道 / 法 / 术入库结果，而是判断：

> 本周哪些输入、行动和反馈真正改变了我的理解、选择和下一步行动；同时保留高质量原材料的锋芒，避免被 AI 综合摘要抹平。

核心目标：

1. 看见本周真正重要的输入、行动和变化；
2. 保留高质量输入源的核心要点；
3. 提炼认知增量、盲点和反例；
4. 检查认知是否进入行动，行动是否获得反馈，反馈是否修正判断；
5. 形成下周最小高价值行动和人工确认候选。

## 2. 判断优先级

生成时遵守：

1. 重要性 > 完整性；
2. 行动证据 > 自我反馈 > 高价值输入 > 一次性信息查询 > 工具噪声；
3. 原料锋芒 > 综合顺滑；
4. 真实反馈 > 内部产物；
5. 长期判断 > 短期情绪；
6. 可读性 > 结构完整；
7. 宁缺毋滥 > 硬凑模块。

来源目录不决定价值。最终判断依据是材料是否改变理解、判断、行动或长期系统。

必须区分：

* `00_log/` 是自我反馈输入，不是 Memory；
* Memory 是人工确认后的跨周上下文；
* AI Chat 精华问题摘要必须作为重要输入源单独关注，保留问题与核心回答的认知密度；
* 但 AI Chat 的影响范围要受控：它不能天然成为本周主线、行动证据、Memory 或道 / 法 / 术候选，除非确实改变了判断、行动或长期问题意识。

## 3. Process Pack 使用边界

`04_output/_dist/weekly/YYYY-Www/process-pack.md` 是 AI 读取材料的中间包，不是最终报告。

AI 必须基于 Process Pack 重新判断、压缩、重排、改写和删除。

不要照搬脚本关键词命中结果。
不要把 `input.json` 作为常规上下文，除非用于来源核查、缺口检查或排错。
不要把 `_dist` 里的候选当作最终判断。

AI 加工分三层：

1. 保留原料核心；
2. 提炼洞察、盲点和长期判断；
3. 转化为行动闭环和人工确认候选。

## 4. 高质量输入源处理

Weekly Output 必须判断是否存在值得单独回顾的高质量输入源，尤其包括：

* Flomo 笔记；
* AI Chat 精华问题摘要；
* 读书笔记；
* 关键会议记录；
* 关键微信聊天；
* 关键行动记录；
* 对长期判断、写作、项目、人生问题有明显价值的原始材料。

重要输入源回顾的目标是保留原材料本身的问题意识、思想密度和独特表达。

不要做来源核对表。
不要把所有文件逐条列出。
不要只写路径。
可以保留极短原文或关键表达，但不要大段摘录。
后续洞察可以基于这些材料加工，但不能用 AI 的综合判断替代原料本身。

## 5. 生成前内部判断

生成前先完成内部判断，不需要输出完整过程：

1. 本周主线是什么？
2. 哪些输入真正改变了判断、行动或长期问题意识？
3. 哪些原材料本身值得保留？
4. 本周是输入为主、判断为主、行动为主，还是反馈修正为主？
5. 有哪些洞察、盲点、反例和未验证假设？
6. 输出应写多密：材料强则展开，材料弱则少写。

洞察最多 3 条，必须能改变理解、解释模式、迁移到未来场景或影响下周行动。

盲点最多 3 条，重点识别：高估、低估、未验证、过度系统化、行动不足、漂亮表达但缺少证据。

## 6. 做中学与行动闭环

Weekly Output 必须评估本周真实行动质量。

重点判断：

* 本周是否真的做了什么；
* 行动是否产生反馈；
* 反馈来自哪里；
* 反馈是否修正判断；
* 下一步最小动作是什么。

做中学评分：

* C：几乎只有输入、聊天、思考，没有真实行动；
* B：有行动或产物，但反馈弱，仍多停留在想法层；
* A：有明确行动，也获得反馈，并修正了部分判断；
* S：行动 → 反馈 → 修正 → 再行动，形成完整学习闭环。

规则：

* 没有第二轮行动或现实反馈，原则上不能评 S；
* 不要把“完成产物”等同于“获得反馈”；
* 不要把“想明白了”等同于“行动闭环”；
* 内部产物只能算系统反馈或内部反馈，不能直接算外部反馈。

反馈类型：

* 外部反馈：他人回复、用户反应、市场数据、真实执行结果，权重最高；
* 系统反馈：脚本运行、工具产物、流程卡点，权重中；
* 内部反馈：自我感受、AI 对话、认知变化，权重低。

## 7. 语言锋芒、金句与认知冲击

Weekly Output 不只要正确，还要能刺中认知。

在非原料摘录部分，尤其是「整体总判断」「本周总览」「核心洞察」「盲点与反例」「做中学评分」「下周行动清单」中，AI 必须为重要内容生成精炼、有冲击力、能触发反思的判断句。

这类句子不是装饰，而是认知钉子。

### 7.1 金句标准

重要判断应尽量压缩成金句。金句必须满足：

1. **自闭环**：单独看也能成立，不依赖大段解释；
2. **短而硬**：优先 8-20 字，少用长句；
3. **有反转**：刺破一个默认假设、舒适叙事或漂亮幻觉；
4. **有代价感**：让用户看到继续这样做会损失什么；
5. **有行动指向**：读完后能影响判断、删除、选择或行动；
6. **受证据约束**：不能为了好听而夸张、玄学或编造。

金句不是总结的结尾，而是判断的锋刃。

### 7.2 生成方法

对重要内容，AI 按三步处理：

1. 先写出普通判断：这件事真正说明什么？
2. 再找反转关系：用户可能误以为什么？现实可能相反在哪里？
3. 最后压缩成金句：删解释、删铺垫、保留最锋利的判断。

可用句式：

* 反转句：你以为是 A，其实是 B。
* 边界句：没有 X，Y 只是 Z。
* 代价句：继续 X，代价是 Y。
* 审判句：判断 X，只看 Y。
* 隐喻句：A 是剑，也可能是茧。
* 截断句：不要再优化 X，先验证 Y。

每个关键模块至少保留 1 句金句，但不要堆砌。
普通内容不强行写金句，重要内容必须有锋芒。

### 7.3 示例风格

示例不是固定模板，只代表强度标准。AI 应基于当周真实材料自行生成，不要机械套用。

* 反转思考：Learn-X 会成为你的茧。
* 没有现实回声，深度只是回音壁。
* 高质量输入不行动，只是高级消费。
* 你不是在优化系统，你是在推迟现实。
* 周报不是为了记住更多，而是为了删掉幻觉。

### 7.4 使用边界

金句只用于重要判断，不用于填充篇幅。

禁止：

* 为了冲击而夸张；
* 为了押韵而牺牲准确；
* 为了金句而制造无证据判断；
* 把普通内容包装成深刻；
* 把情绪判断伪装成长期原则。

原料摘录部分优先保留原文锋芒，不强行改写。
AI 加工部分必须比普通摘要更锋利，但仍受证据约束。

最终效果：用户读完后，不只是知道本周发生了什么，还能被一句话击中一个盲点。

## 8. 默认输出结构

默认按以下结构输出，可根据材料质量减少，不硬凑：

0. 整体总判断：1-3 条，判断本周主线；
1. 本周总览：3-5 条，说明真正重要的变化；
2. 重要输入源回顾：最多 3-5 个，重点保留高质量原料，尤其是 AI Chat 精华问题摘要；
3. 核心洞察：最多 3 条；
4. 盲点与反例：最多 3 条；
5. 做中学评分：说明评分、证据、反馈类型和判断边界；
6. 下周最小高价值行动：最多 3 项，每项有完成边界和价值点；
7. 人工确认清单；
8. 来源；
9. 芒格之魂的洞察：仅保留标题和 todo，用于用户后续手动补充基于原始数据的洞察。

## 9. 人工确认清单

这是 Weekly Output 之后进入 Memory / Core 的唯一默认勾选区。

AI 只能提出候选，不能替用户确认、入库或改写 `01_core/道/`、`01_core/法/`、`01_core/术/`。

默认全部使用 `- [ ]`，由用户手动改成 `- [x]`。

### 9.1 Memory 候选

用于沉积到 `01_core/memory/YYYY-QN.memory.md` 的极薄跨周上下文。

只保留本周已经比较确认、下周仍有复用价值的判断。
每条 20-40 字，不写解释，不写过程。候选 5-10 条，宁缺毋滥。

格式：

* [ ] Memory：
* [ ] Memory：

没有强信号时写：

> 本周不建议生成新的 Memory 候选。

### 9.2 继续追踪

用于保留尚未闭环、但值得下周继续观察的问题。

它不是 Memory，也不是行动任务，而是跨周观察线索。

每周最多 3 条。

格式：

* [ ] 继续追踪：
* [ ] 继续追踪：

### 9.3 道 / 法 / 术观察候选

这里只输出观察候选，不代表已入库。

* 道：长期价值判断、人生方向、主体性原则；
* 法：可迁移的判断框架；
* 术：可执行方法、流程、模板、复盘动作。

规则：

* 道默认 0 条，除非有长期稳定、可反复指导选择的新证据；
* 法最多 2 条，必须可迁移；
* 术最多 3 条，必须能被下周行动验证；
* 每条不超过 50 字；
* 长期反复出现但本周没有新证据的判断，不重复包装为新沉淀。

格式：

* [ ] 道候选：
* [ ] 法候选：
* [ ] 术候选：

没有强信号时写：

> 本周不建议沉淀新的道 / 法 / 术候选。

## 10. 证据与来源

Weekly Output 必须可靠，不编造输入中没有的信息。
如果判断来自推断，要标注为“观察”“推测”或“待确认”。

重要判断要可追溯，但不要让正文被路径打断。

推荐方式：

* 正文使用 source id 或简短来源说明；
* 末尾统一汇总最小必要来源路径；
* 只保留支撑关键判断的来源，不堆砌路径。

## 11. 质量标准与禁止项

Weekly Output 必须可靠、重要、可读、有摘要、有原料、有洞察、有盲点、有行动、有来源。

避免：

* 把脚本关键词命中结果当最终判断；
* 把重要输入源写成路径清单；
* 把低质量来源硬塞进重要输入源；
* 把未验证的漂亮句子写成道 / 法 / 术；
* 把旧判断重复包装成新沉淀；
* 把内部产物误判成外部反馈；
* 为了长期视角写空泛哲学；
* 为了语言冲击制造夸张结论；
* 固定输出图谱、第一性原理、Prompt、Skill、写作或 Demo 候选。

输入质量不足时，诚实少写，说明缺口，不硬凑模块。

生成后内部自检：

1. 是否有摘要、原料、洞察、盲点、行动、来源；
2. 是否保留高质量原料锋芒；
3. 是否误判行动和反馈；
4. 是否删除空话、重复话和无证据判断；
5. 关键 AI 总结部分是否足够精炼、有力、能触动认知。
