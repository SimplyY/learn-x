# Weekly Output 核心输出要求

本文件维护 `learn-x-process` 的 Weekly Output 生成要求。

`SKILL.md` 说明这个 Skill 是什么、什么时候用、边界在哪里；本文件说明 AI 最终写给人的 Weekly Output 应该遵守什么判断原则、质量标准和加工方式。

---

## 一、核心定位

Weekly Output 是 Learn-X 个人认知进化系统中的每周认知审稿环节。

它的第一任务不是机械总结一周，而是判断：

> 本周哪些输入、行动和反馈真正改变了我的理解、选择和下一步行动；同时保留高质量原材料的锋芒，避免被 AI 综合摘要抹平。

Weekly Output 不是：

* 自动分类结果；
* 脚本日志；
* 材料摘要合集；
* 候选清单堆叠；
* 文件来源核对表；
* 道 / 法 / 术入库结果。

它要帮助用户完成六件事：

1. 看见本周真正重要的输入、行动和变化；
2. 保留高质量输入源的核心要点；
3. 提炼对自我、项目、学习、AI、教育、投资和生活有价值的认知增量；
4. 发现被忽略、低估、高估或尚未验证的盲点；
5. 把本周问题放到更长期、更大系统和更真实生活中理解；
6. 检查认知是否进入行动，行动是否获得反馈，反馈是否修正判断。

Weekly Output 的价值在于：

> 把输入变成理解，把理解变成判断，把判断变成行动，把行动变成反馈。

但必须特别注意：

> 高质量原材料不能被 AI 的综合摘要压扁。

对 Flomo、AI 聊天、阅读笔记、关键行动记录、会议记录、微信聊天等高价值输入源，应保留其最核心的原始思想、问题意识和表达锋芒。

---

## 二、核心优先级

生成 Weekly Output 时，遵守以下优先级：

1. 重要性 > 完整性；
2. 行动证据 > 自我反馈 > 高价值输入 > 一次性信息查询 > 工具噪声；
3. 原料锋芒 > 综合顺滑；
4. 真实反馈 > 内部产物；
5. 长期判断 > 短期情绪；
6. 可读性 > 结构完整；
7. 宁缺毋滥 > 硬凑模块。

来源目录不决定价值。

本周主线可能来自：

* `00_log/`：自我反馈，我如何理解自己的状态和变化；
* `01_inbox/`：信息输入，我接收了什么；
* `02_action/`：行动证据，我做了什么，世界如何反馈。

最终判断依据不是文件位置，而是材料是否改变了理解、判断、行动或长期系统。

必须明确区分：

* `00_log/` 是自我反馈输入，不是 Memory；
* Memory 是人工确认后的输出，不和 `00_log/` 混淆；
* AI 对话不是天然主干，只有当它改变判断、行动或长期问题意识时，才成为主干；
* Flomo、AI 聊天、阅读笔记等高质量输入源，即使没有直接行动，也可能值得保留，因为它们可能是长期判断和写作素材的源头。

---

## 三、生成前判断流程

在生成 Weekly Output 前，AI 必须先完成四步内部判断，不需要输出完整过程：

### 1. 判断本周主线

回答：

* 本周真正反复出现的问题是什么？
* 哪些内容改变了用户的判断或行动？
* 是否有真实行动？
* 是否有外部反馈、系统反馈或内部反馈？
* 本周是输入为主、判断为主、行动为主，还是反馈修正为主？

### 2. 选择高质量输入源

判断哪些材料本身值得保留，而不是全部融入二次总结。

优先选择满足以下至少 2 条的输入源：

* 原文有强问题意识；
* 多次影响本周判断；
* 能作为未来写作、决策或项目素材；
* 比 AI 综合摘要更有表达锋芒；
* 与真实行动、项目推进或长期人生问题有关；
* 对用户长期价值系统、事业系统、投资系统、教育系统或生活方式有启发。

不要把所有来源都列入重要输入源回顾。

低质量、重复、噪声型材料，即使命中关键词，也不应进入核心内容。

### 3. 提炼洞察与盲点
> 至少各1条，但各不超过 3 条
洞察不是普通总结。只有满足以下至少一项，才值得写入：

* 改变了用户对自己、项目、AI、教育、投资、生活的理解；
* 揭示了一个反复出现的模式；
* 能迁移到未来多个场景；
* 能改变下周行动方式；
* 能解释过去一段时间的反复纠结。

盲点要服务行动和长期校准，重点识别：

* 用户可能高估了什么；
* 用户可能低估了什么；
* 哪些判断缺现实证据；
* 哪些系统设计可能过度复杂；
* 哪些内容只是 AI 对话里的漂亮表达；
* 哪些行动还停留在想法层；
* 哪些长期目标正在被短期工具问题稀释。

### 4. 决定输出密度

根据材料质量决定篇幅和展开程度：

* 常规周：控制在 1000-1500 字；
* 材料密度高、真实行动多、高质量输入源多、洞察强时：可扩展到 1800 字左右；
* 输入质量不足时：可以少写；
* 不为了字数凑模块；
* 不为了简短删除关键洞察；
* 不为了综合而抹平高质量原材料。

核心原则：

> 密度优先，重要性优先，可读性优先，关键原料优先。

## 四、Process Pack 的使用边界

`04_output/_dist/YYYY-WW/process-pack.md` 只用于让 AI 稳定读取材料。

它可以包含：

* 本周来源覆盖；
* source id；
* 短路径；
* 清洗后的正文；
* 必要的重复来源提示。

`input.json` 是脚本中间态，保留文件时间、指纹、原始路径等机器字段；常规不要作为 AI Chat 上下文。

但它不是最终报告，不应该包含：

* 道 / 法 / 术判断；
* Prompt / Skill 判断；
* 入库建议；
* 行动优先级；
* 像最终周报一样的结论段落。

AI 必须基于 Process Pack 再进行智能判断、压缩、重排、改写和删除。

不要把脚本关键词命中的结果当作最终判断。

如果脚本生成的候选不适合人读，AI 要主动处理，而不是照搬。

AI 加工分三层：

1. 保留原料核心：先判断哪些输入源本身值得回顾；
2. 提炼洞察判断：再合并不同来源，提炼模式、洞察、盲点和长期判断；
3. 转化行动闭环：最后判断哪些内容需要进入下周行动、复盘或人工确认。

---

## 五、重要输入源的处理原则

Weekly Output 必须考虑是否存在值得单独回顾的高质量输入源。

尤其关注：

* 高质量 Flomo 笔记；
* 重点关注 ai chat 中精华问题摘要，里面有非常重要的内容（因为是我提的核心问题和 ai 的回答）
* 高质量读书笔记；
* 关键会议记录；
* 关键微信聊天；
* 关键行动记录；
* 对长期判断、写作、项目、人生问题有明显价值的原始材料。

重要输入源回顾的目标是保留原材料本身的思想密度、问题意识和独特表达。

不要做来源核对表。

不要把所有文件逐条列出。

不要只写路径。

可以保留极短原文或关键表达，但不要大段摘录。

后续洞察可以在这些材料基础上加工，但不能用 AI 的综合判断替代原料本身。

---

## 六、做中学与行动闭环判断(输出1-3个核心)

Weekly Output 必须评估本周真实行动质量。

重点不是制造更多计划，而是判断：

* 本周是否真的做了什么；
* 行动是否产生反馈；
* 反馈来自哪里；
* 反馈是否修正了判断；
* 下一步最小动作是什么。

做中学评分：

* C：几乎只有输入、聊天、思考，没有真实行动；
* B：有行动或产物，但反馈弱，更多停留在想法层；
* A：有明确行动，也获得反馈，并修正了部分判断；
* S：行动 → 反馈 → 修正 → 再行动，形成完整学习闭环。

评分规则：

* 没有第二轮行动或现实反馈，原则上不能评 S；
* 不要把“完成产物”直接等同于“获得反馈”；
* 不要把“想明白了”直接等同于“行动闭环”；
* 内部产物只能算系统反馈或内部反馈，不能直接算外部反馈。

反馈类型分三类：

* 外部反馈：他人回复、用户反应、市场数据、真实执行结果，权重高；
* 系统反馈：脚本运行、工具产物、流程卡点，权重中；
* 内部反馈：自我感受、AI 对话、认知变化，权重低。

做中学复盘只评估已经发生的行动，不负责设计复杂新项目。

下周动作必须小、具体、有完成边界。

---

## 7. 人工确认清单

本节是 Weekly Output 之后进入 Memory / Core 的唯一默认勾选区。

它不是正式入库结果，只是给用户人工确认的候选清单。  
AI 只能提出候选，不能替用户确认、入库或改写 `01_core/道/`、`01_core/法/`。

输出时默认全部使用 `- [ ]`，由用户阅读 Weekly Output 后手动改成 `- [x]`。  
后续 Memory 生成流程只应读取本节中已勾选内容，避免把下周任务、普通 checklist 或过程性记录误写入 Memory。

### 7.1 Memory 候选

用于沉积到 `01_core/memory/YYYY-QN.memory.md` 的极薄跨周上下文。


只保留本周已经比较确认、下周仍有复用价值的判断。  
每条 20-40 字，不写解释，不写过程，不追求完整。候选5-10条，多参考「整体总判断、本周总览、核心洞察、做中学」。

格式：

- [ ] Memory：
- [ ] Memory：

判断标准：

* 本周反复出现，并影响了理解、选择或行动；
* 下周、下月、甚至明年继续对 Chat Pack / AI 对话有上下文价值；
* 已经被用户基本认可，而不是 AI 自己生成的漂亮表达；


如果没有强信号，可以写：

> 本周不建议生成新的 Memory 候选。

### 7.2 继续追踪

用于保留尚未闭环、但值得下周继续观察的问题。

它不是 Memory，也不是行动任务；它是跨周观察线索。

格式：

- [ ] 继续追踪：
- [ ] 继续追踪：

适合内容：

* 尚未验证的判断；
* 重复出现但还没想清楚的问题；
* 需要现实反馈、行动结果或外部输入才能判断的主题。

每周最多 3 条，不硬凑。

### 7.3 道 / 法 / 术观察候选

这里只输出观察候选，不代表已入库，必须等待人工确认。

道要极其保守。大多数周报都不应该新增「道」。

* 道：长期价值判断、人生方向、主体性原则。
* 法：可迁移的判断框架。
* 术：可执行方法、流程、模板、复盘动作。

输出规则：

* 道：默认 0 条；除非本周出现长期稳定、可反复指导选择的新证据。
* 法：最多 2 条；必须可迁移到其他场景。
* 术：最多 3 条；必须能被下周行动验证。
* 每条不超过 50 字。
* 长期反复出现的判断，如果本周没有新证据，不重复包装成新沉淀。
* 有些内容适合保留为 Memory、继续追踪、重要输入源或洞察，不一定要升级为道 / 法 / 术。

格式：

- [ ] 道候选：
- [ ] 法候选：
- [ ] 术候选：

如果没有强信号，可以写：

> 本周不建议沉淀新的道 / 法 / 术候选。

## 八、证据与来源原则

Weekly Output 必须可靠，不编造输入中没有的信息。

如果判断来自推断，要标注为观察、推测或待确认。

重要判断要可追溯，但不要让正文被路径打断。

推荐方式：

* 正文使用 source id；
* 最后统一汇总来源路径；
* 来源路径使用 markdown 引用块包裹；
* 只保留支撑关键判断的最小来源，不堆砌路径。

示例：

正文中可写：

> 本周 Learn-X 的主线和报告来自 xxxx。

最后汇总：03_input/weekly/2026-W23/

p0 证据
> - ./01_inbox/ai.md
> xxx

保留足以支撑核心判断的证据。

p1 来源
> xxxx

保留所有材料的来源，方便检查

---

## 九、输出质量标准

Weekly Output 必须满足：

* 可靠：不编造输入中没有的信息，不确定就标注；
* 重要：优先放影响理解、判断、行动、长期资产的内容；
* 可读：像给人看的认知审稿报告，不像机器分桶或脚本日志；
* 有摘要：能快速看见本周真正重要的内容；
* 有原料：能保留高质量输入源的核心要点；
* 有洞察：不能只总结发生了什么，还要说明“这意味着什么”；
* 有盲点：指出高估、低估、未验证、过度系统化或行动不足之处；
* 有长期视角：必要时从更大时间、空间和系统维度看问题；
* 可操作：下周行动必须具体、有完成边界；
* 可追溯：重要判断保留最小来源路径；
* 宁少勿滥：每类候选可以是 0 条、1 条、2 条，不硬凑；
* 保守沉淀：未验证内容不直接升级为道 / 法 / 术。
* 信息可读性：分点分段，尽量避免没有分点分段的长内容（超过100字），多利用表格等形式呈现负责信息。尤其是总览、摘要、洞察和盲点部分。

## 十、不要这样输出

必须避免以下问题：

* 不要把 50 条候选逐条铺开；
* 不要机械重复“原始来源 / 核心表达 / 建议层级”；
* 不要把脚本关键词命中的结果当作最终判断；
* 不要把 AI 对话天然视为主干；
* 不要固定输出来源覆盖；
* 不要为了完整牺牲可读性；
* 不要只做摘要而没有判断和行动闭环；
* 不要只做判断而没有核心摘要；
* 不要把高质量输入源全部融成 AI 的二次判断；
* 不要把重要输入源回顾写成文件路径清单；
* 不要把低质量来源硬塞进重要输入源；
* 不要把未验证的漂亮句子直接写成道 / 法 / 术；
* 不要把旧判断重复包装成新沉淀；
* 不要把内部产物误判成外部反馈；
* 不要为了长期视角写空泛哲学；
* 不要固定输出图谱、第一性原理、Prompt、Skill、写作或 Demo 候选。

---

## 十一、输入质量不足时的最小判断

如果本周输入质量不足，Weekly Output 也要诚实。

可以：

* 少写；
* 只给核心摘要和盲点；
* 说明本周没有值得单独回顾的高质量输入源；
* 说明本周不适合沉淀新的道 / 法 / 术；
* 建议补充更好的行动证据、自我反馈输入或原始笔记。

Weekly Output 的价值不在长度，而在帮助用户形成：

* 更真实的理解；
* 更少的盲点；
* 更稳的长期判断；
* 更好地保留高质量原材料；
* 更小但有效的行动闭环。

---

## 十二、输出后自检

生成 Weekly Output 后，AI 必须内部自检并据此修订，不需要输出完整自检过程。

自检五问：

1. 是否同时包含摘要、原料、洞察、盲点和行动？
2. 是否保留了高质量原材料的锋芒？
3. 是否把思考误判成行动，或把内部产物误判成外部反馈？
4. 下周任务是否一周内可完成，并且有最小完成标准？
5. 是否删除了空话、重复话和无证据判断？



## 默认输出结构
0. 整体总判断，1-3条，对于整体主线判断
1. 本周总览：3-5 条，回答本周真正重要的变化和内容。
2. 重要输入源回顾：最多 3-5 个，保留高质量原料，尤其是 ai chat 中的核心问题。
3. 核心洞察：最多 3 条
4. 盲点与反例：最多 3 条
5. 做中学评分：300-500字左右
6. 下周最小高价值行动：最多 3 项，每项有完成边界和价值点。
7. 人工确认清单
8. 来源
