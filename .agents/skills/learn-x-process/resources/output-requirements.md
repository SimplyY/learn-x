# Weekly Output 输出要求

本文件维护 `learn-x-process` 的核心输出要求。

`SKILL.md` 说明这个 Skill 是什么、什么时候用、边界在哪里；本文件说明最终写给人的 Weekly Output 应该长什么样。

## 核心定位

Weekly Output 不是自动分类结果，也不是脚本日志。

它是给人一字不落阅读的每周审稿台，目标是帮助用户做三件事：

1. 看见本周真正反复出现且最重要的问题。
2. 找到少量值得人工确认、行动或沉淀的内容。
3. 保持报告精简，只保留本周最重要的判断。
4. 检查认知是否进入行动闭环。

## 输入来源平等

不要预设主干一定来自 `03_input/inbox/ai/`。

本周主线可能来自：

- `03_input/inbox/`：信息输入，我接收了什么。
- `03_input/action/`：行动证据，我做了什么、世界如何反馈。
- `03_input/log/`：自我反馈，我如何理解自己的状态和变化。

判断主线时看内容质量、重复频率、现实重要性和行动价值，而不是看来源目录。

Weekly Output 必须明确区分：

- index 只是处理清单，不是周记；
- log 是自我反馈输入，不是 Memory；
- Memory 是人工确认后的输出，不和 log 混淆。

## AI 智能责任

脚本只负责确定性工作：

- 读取文件
- 解析格式
- 清洗文本
- 合并与去重
- 编号
- 统计来源覆盖
- 生成 Input Pack 和 Process Pack

最终 Weekly Output 必须由 Codex 按本文件进行智能判断和改写。不要把脚本输出原样当作最终报告。

如果脚本生成的候选不适合人读，Codex 要主动压缩、重排、改写、删除，而不是照搬。

## Process Pack 不是报告

`04_output/_dist/YYYY-WW/process-pack.md` 只用于让 AI 稳定读取材料。它可以包含：

- 本周来源覆盖
- 每条材料的 source id
- 原始路径
- 文件时间
- 清洗后的正文片段
- 重复信息

它不应该包含：

- 道 / 法 / 术 判断
- Prompt / Skill 判断
- 入库建议
- 行动优先级
- 像最终周报一样的结论段落

## 输出质量标准

Weekly Output 必须满足：

- **可靠**：不编造输入中没有的信息；不确定就标注观察或待确认。
- **重要**：优先放真正影响判断、行动、长期资产的内容。
- **可读**：用户能顺着读下去，不像机器分桶或日志。
- **可操作**：做中学复盘和下周行动必须具体。
- **宁少勿滥**：每类候选可以是 0 条、1 条、2 条，不要硬凑。
- **可追溯**：重要候选保留来源路径。
- **精简**：输出字数控制在 1000 字以内（不重要的内容、板块、标题、可以省略）

## 做中学要求

Weekly Output 不只是总结与归类，还要检查本周判断是否进入行动闭环。

本段只做行动复盘和必要判断，不自动生成正式长期资产，不替代人工审核。


## Weekly Output：做中学复盘

识别本周我已经做过的真实且关键的行动、交流、实验、写作、Demo、调研或作品，并判断它们是否真正促进了学习。

重点不是新增更多行动，而是先评估本周存量行动的质量。

### 输出要求

请先给本周「做中学评分」：

- C：几乎只有输入、聊天、思考，没有真实行动。
- B：有少量行动，但反馈弱，更多停留在想法层。
- A：有明确行动，也获得了一些现实反馈。
- S：行动与反馈形成了学习闭环，明显推动了判断或能力进化。

### 识别内容

优先从本周材料中提取：

- 我实际做了什么；
- 哪些行动带来了反馈；
- 哪些判断被验证、修正或暴露问题；
- 哪些事情值得继续做得更好；
- 哪些只是聊得很多、但行动不足。
- 哪些判断仍未进入行动，需要下周补最小动作。

### 输出格式

### 做中学复盘
#### 主题标题 20-30 字

- 本周真实行动：
- 解决的问题 & 获得的反馈：
- 核心洞察：
- 当前评分：
- 下一步优化：继续做得更好 / 补一个最小动作 / 暂停观察

判断原则：

- 优先看已经发生的行动，而不是制造新计划。
- 如果本周已经做了核心事情，重点是如何提高质量。
- 如果行动不足，再提出一个最小补充动作。
- 行动闭环检查并入做中学复盘，不单独开固定模块。
- 不要输出庞大计划。

## 推荐结构

### 0. 本周总览
> 写上精准的输出时间（生成时间即可，不应该包含周范围）

用 50-100 字说明本周真正核心发生了什么。不要只是列主题词。最好能分点分段回答。

应该回答：

- 本周输入的主线是什么？
- 哪个问题反复出现？
- 有没有明显变化、转向或新的张力？
- 哪些来源贡献最大？

### 1. 本周反复出现的核心问题

保留 2-3 个问题。每个问题包含：

- 问题标题
- 100 字解释（最好分段、分点回答）
- 来源线索（需要用md 的引用块 > 包裹）

### 2. 做中学复盘

这是 Weekly Output 的核心部分之一，并承担行动闭环检查。优先评估已经发生的真实行动和反馈，不要制造庞大计划。

使用 `Weekly Output：做中学复盘` 中的评分和格式。

可在本节末尾加入 2-4 条「闭环检查」checkbox，用于确认哪些判断已经行动、哪些仍缺反馈。不要单独输出「行动闭环检查」大节。

### 3. 下周最该做的 3 件事
使用 Markdown checkbox。

只给 3 件。

每件要能在一周内推进，避免抽象口号。

### 4. 值得沉淀的道 / 法 / 术
使用 Markdown checkbox。

道要最少，必须保守。

- 道：长期价值判断、人生方向、主体性原则。
- 法：可迁移的判断框架。
- 术：可执行方法、流程、模板、复盘动作。

输出的时候要注意，最多输出50字描述，精炼易读

## 必要时才提

以下能力不作为 Weekly Output 固定模块：

- 图谱候选
- 第一性原理候选
- 写作 / Demo 候选
- Prompt 候选
- Skill 候选

只有当本周材料中出现强信号，且它们能直接服务行动闭环时，才可以在「做中学复盘」「下周最该做的 3 件事」或「道 / 法 / 术」中用 1 条简短说明带过。不要为它们单独开固定 section。

## 不要这样输出

- 不要把 50 条候选逐条铺开。
- 不要机械重复“原始来源 / 核心表达 / 建议层级”。
- 不要把脚本关键词命中的结果当作最终判断。
- 不要把 AI 对话天然视为主干。
- 不要固定输出「观察、降权或删除」。
- 不要固定输出「来源覆盖」。
- 不要为了完整而牺牲可读性。
- 不要把未验证的漂亮句子直接写成「道」。
- 不要把做中学复盘写成大论文、课程设计或复杂项目计划。
- 不要固定输出图谱、第一性原理、Prompt、Skill、写作或 Demo 候选。

## 最小判断

如果本周输入质量不足，输出也要诚实：

- 可以少写。
- 可以说“本周不适合沉淀道”。
- 可以只给观察清单。
- 可以建议补充更好的输入。

Weekly Output 的价值不在长度，而在帮助用户做更少、更准、更真实的判断。
