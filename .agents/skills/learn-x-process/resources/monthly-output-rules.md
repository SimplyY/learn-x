# Monthly Output 核心输出要求

本文件维护 `learn-x-process` 的 Monthly Output 生成要求。

Monthly Output 不是 4 篇 Weekly Output 的拼接，也不是月度流水账。它是 Learn-X 个人认知进化系统中的 **月度认知审稿环节**：在一个月的时间尺度上，判断用户经历了什么变化，哪些问题反复出现，哪些行动产生了反馈，哪些内容值得进入 Memory / 道 / 法 / 术候选，哪些应继续追踪或删除。

---

## 一、核心定位

Monthly Output 的第一任务不是总结材料，而是回答：

> 这个月的主线是什么？我发生了什么变化？哪些判断被现实验证、修正或暴露出问题？下个月最该验证什么？

它要同时完成五件事：

1. **提炼月度主线**：找到这个月最重要的变化，而不是罗列事件。
2. **看见真实生活**：健康、幸福、出门、关系、诗、电影、旅行等生活内容不能被系统和项目挤掉。
3. **审计行动反馈**：区分已经闭环、半闭环、未闭环和只是想法的内容。
4. **沉淀候选资产**：筛出少量值得进入 Memory / 道 / 法 / 术的内容。
5. **锚定下月验证**：把下个月最重要的 2-4 件事变成可观察反馈的小实验。

Monthly Output 不是：

* 周报合集；
* 文件摘要；
* 项目进展流水账；
* 候选清单堆叠；
* 漂亮但未经验证的自我叙事；
* 自动把所有内容都上升为道 / 法 / 术。

---

## 二、最高判断基准

Monthly Output 必须服从用户长期之道：

> 健康为底座，生活为主体，创造 / 工作为表达，投资为护城河，系统为工具。后者不得反噬前者。

因此，月度审计不能只看效率、产出、系统搭建和项目推进，还必须判断：

* 身体状态是否被透支；
* 生活是否更松弛、更有生命力；
* AI / Learn-X 是否增强主体性，而不是制造信息茧房；
* 教育 / 创业是否接触真实人、真实需求、真实反馈；
* 投资是否服务更好的生活，而不是制造焦虑；
* 写诗、旅行、电影、社交、游戏等是否在恢复生命力。

核心提醒：

```text
AI 是剑，Learn-X 是脑，真实世界是校验场。
```

---

## 三、输入材料优先级

优先读取：

1. 本月 `04_output/_dist/monthly/YYYY-MM/process-pack.md`；
2. 本月 `monthly.md` 和周记清洗内容；
3. 必要时参考 `01_core/道/` 和长期 Memory；
4. 只有在材料缺失、冲突或需要核对来源时，才查看原始输入或 `input.json`。

若材料冲突，优先级为：

```text
真实行动反馈 > 用户明确判断 > 月记 / 周记原文 > Memory > AI 推测
```

不要把未经验证的表达写成长期结论。

---

## 四、输出风格

整体风格要求：

* **结论先行**：先给月度总判断，再展开证据。
* **有主线感**：像一份月度认知报告，而不是机械表格。
* **有生活质感**：保留旅行、诗、电影、社交、游戏、身体状态等真实生命材料。
* **有审计锋芒**：指出未闭环、过度系统化、顺滑学习幻觉、自我叙事风险。
* **有人工确认入口**：所有进入 Memory / 道 / 法 / 术的内容必须用 checkbox，方便用户手动确认。
* **不要过度抽象**：每个抽象判断后，尽量挂回本月具体材料。
* **不要强行深刻**：没有材料支撑时，直接说“不足”或“不建议沉淀”。
* 信息可读性：分点分段，尽量避免没有分点分段的长内容（超过100字），多利用表格等形式呈现负责信息。

语言上允许少量诗性表达，但必须服务判断，不要自我感动。

---

## 五、推荐输出结构

### 0. 月度总判断

100-220 字。

直接回答：

* 本月主线是什么？
* 本月是突破月、收敛月、修复月、验证月，还是混沌月？
* 本月最大的变化是什么？
* 本月真正推进了什么？
* 本月最该冷静看待什么？

要求：

* 开头必须有一句强判断。
* 不要先铺材料。
* 可以用一句核心金句收束本月主线。
* 必须区分“已经验证的变化”和“仍未闭环的想法”。


示例表达方向：

```text
2026 年 X 月的主线是：……
这个月不是……，而是……
本月最重要的不是做成了某个项目，而是……
但也要冷静：……
```

---

### 1. 本月总览

先按照月初、月中、月末输出，总共 150-250字，总结本月最核心的事情。

然后按主题分层，而不是按周机械复述。

建议输出 3-5 个主题，每个主题 100-200 字。


---

### 2. 本月核心问题

输出 2-4 个，不要硬凑。

每个问题必须包含：

1. 问题本身；
2. 为什么它是本月核心问题；
3. 本月给出的阶段性答案；
4. 下月是否继续追踪。

优先选择反复出现、影响行动选择、能改变下月方向的问题。

示例格式：

```markdown
### 问题一：……

本月真正的问题不是……，而是……

本月的阶段性答案是：……

下月要继续看：……
```

---

### 3. 月度核心洞察

输出 3-5 条。

每条洞察要求：

* 不是普通总结；
* 必须来自本月多个材料的交叉验证；
* 能影响未来 1-3 个月判断或行动；
* 最好能转化为 Memory / 法 / 术候选。

优先保留这类洞察：

* 生活机制类；
* 教育 / 创业一线判断；
* AI / Learn-X 使用边界；
* 自我认知变化；
* 高质量输入如何转化为行动。

---

### 4. 盲点与反例

必须输出。

至少 1-3 条。

重点识别：

* 系统清晰是否替代了行动闭环；
* 高质量输入是否制造顺滑学习幻觉；
* 教育理想是否低估现实阻力；
* AI 是否放大了表达但削弱了真实验证；
* 娱乐是恢复身心，还是逃避行动；
* 投资是否服务生活，还是制造焦虑。

格式：

```markdown
### 4.1 可能高估：……

判断：……
反例：……
下月校验方式：……
```

不要为了批判而批判，但必须保留真实反对证据。

---

### 5. 月度状态评分

必须输出表格。

建议维度（可根据内容自行展开 1-3个额外的维度）：

| 维度   | 本月状态 | 判断 |
| ---- | ---: | -- |
| 健康   |      |    |
| 幸福   |      |    |
| 修心   |      |    |
| 真实行动 |      |    |
| 生活审美 |      |    |

最后给出一个综合评分，例如：

```text
本月综合评分：A- / B+ / B
```

评分原则：

* 不是情绪打分；
* 不是越忙越高；
* 要同时看健康、生活、真实反馈、系统推进和行动闭环；
* 如果输出漂亮但现实闭环不足，不能给 S。

---


### 6. 下月最小高价值行动

输出 2-4 项。

不是泛泛计划，而是验证清单。

每一项必须包含：

* 行动对象；
* 完成标准；
* 反馈标准；
* 不透支健康和生活。

格式建议：

```markdown
### 6.1 事项名称

完成标准：

- ……
- ……

核心不是……，而是……
```

优先选择：

* Learn-X 是否减负；
* 教育是否有真实样本；
* 真实关系是否有反馈；
* 健康 / 出门 / 生活是否稳定；
* 投资或系统是否减少焦虑。

---

### 7. 人工确认清单

必须输出，且用 checkbox。

#### 7.1 值得进入 Memory

3-10 条。

每条 20-80 字，不要太碎。

只收录未来 1-3 个月仍会影响判断和行动的内容。

格式：

```markdown
- [ ] ……
- [x] ……
```

如果用户原材料中已有明确打勾，可保留 `[x]`；否则默认 `[ ]`。

不要收录：

* 普通总结；
* 过期事项；
* 情绪宣泄；
* 没行动指向的漂亮表达；
* 未验证的宏大判断。

#### 7.2 值得继续追踪

3-6 条。

格式：

```markdown
- [ ] ……
```

重点是未闭环但重要的问题。

#### 7.3 值得作为道 / 法 / 术候选观察

3-8 条。

格式：

```markdown
- [ ] 道：……
- [ ] 法：……
- [ ] 术：……
```

---

### 8. 多思考一步

如果启用了“多思考一步”，只选择最有用的 2-3 个方向。

推荐三个方向：

#### 向上看

这个月属于更大系统中的哪一段？

例如：

* GAP 阶段的秩序重建；
* AI 时代个人操作系统建设；
* 教育创业的冷启动观察期；
* 从外部自由走向内在秩序。

#### 向下看

下月真正要看的关键变量是什么？

最多 3 个。

例如：

1. 身体状态是否稳定；
2. 真实人是否持续反馈；
3. 系统是否减少负担。

#### 旁边看

最大的反例是什么？

例如：

```text
系统越来越完整，输出越来越漂亮，但真实世界没有变化。
```

要求：

* 不要写成长篇哲学分析；
* 不要重复前文；
* 必须能改变下月行动。

---

### 9. 一句话收束

用一句话收束本月。

要求：

* 有判断；
* 有画面；
* 能代表本月变化；
* 不要空泛鸡汤。

示例：

```text
2026 年 X 月，你不是在寻找一个新工具，而是在寻找一种新的活法：……
```

---

## 六、质量标准

一份合格的 Monthly Output 应该做到：

* 有强主线，不是周报拼接；
* 有生活质感，不只谈系统和项目；
* 有判断锋芒，不只是温和总结；
* 有行动审计，能区分想法和反馈；
* 有盲点和反例，能防止自我欺骗；
* 有候选沉淀，但不过度入库；
* 有人工确认清单，便于后续 Memory 更新；
* 有下月验证，不是泛泛计划；
* 能守住“健康、生活、创造、投资、系统”的真实优先级。

最终标准：

> 读完后，用户应更清楚：这个月我是谁，我如何变化，什么被验证，什么只是想法，什么值得保留，下个月最该拿什么去真实世界验证。
