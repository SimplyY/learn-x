# Memory 输出规则

Memory 是人工确认后的跨期上下文，不是 Output 摘要，也不是正式 `道/`、`法/`、`术` 入库结论。

## 核心原则

人工勾选内容优先无损迁移。

- 已勾选 checkbox：必须进入 Memory，除非用户明确撤销。
- 不设数量上限；用户勾选多少，Memory 就迁移多少。
- 不做有损压缩，不改写关键限定、比喻、因果、对象和判断。
- 允许无损整理：去掉 checkbox 标记、统一 bullet、归入小节、删除完全重复项。
- 未勾选内容默认不写入；只能作为候选池或继续追踪池。

## 输入

主要读取：

- `04_output/weekly/YYYY-WW.md`
- `04_output/monthly/YYYY-MM.md`
- `04_output/yearly/YYYY.md`
- 对应 `_dist/**/memory-candidates.md`

重点看：

- 已勾选 checkbox：`- [x]`、`* [x]`
- 明确标记为「进入记忆」「继续追踪」「重要」「保留」「确认」的内容
- 用户当场明确要求保留的内容

## 输出位置

Weekly / Monthly 写入季度文件：

```text
01_core/memory/YYYY-QN.memory.md
```

Yearly 写入年度文件：

```text
01_core/memory/YYYY.memory.md
```

小节格式：

```md
## 候选观察池
## YYYY-Www
## Monthly｜YYYY-MM
## Yearly｜YYYY
```

## 推荐结构

候选观察统一放在季度 / 年度 Memory 文件顶部，不分散维护在具体周、月小节中：

```md
## 候选观察池

### 道候选

- [2026-W22] 道：……
- [Monthly｜2026-05] 道：……

### 法候选

- [2026-W22] 法候选：……

### 术候选

- [Monthly｜2026-05] 术：……
```

周 / 月 / 年小节只保留 Memory 和继续追踪：

```md
## YYYY-Www

### Memory

- Memory：……

### 继续追踪

- 继续追踪：……
```

月 / 年同理，按来源候选区分为：

- 值得进入 Memory
- 值得继续追踪

`值得作为道 / 法 / 术候选观察` 不放在具体周期小节里，统一迁移到顶部候选观察池，并保留来源标签。

## 候选观察池维护

候选观察池是季度级候选审查区，不是正式长期资产。

写入规则：

- 已勾选的道 / 法 / 术候选观察必须进入候选观察池。
- 条目前必须保留来源，例如 `[2026-W22]`、`[Monthly｜2026-05]`。
- 原文无损迁移，不改写候选判断。
- 同一候选跨周 / 跨月重复出现时，可以在同一条后追加来源，不删除语义差异。

季度审查：

- 每个季度结束时，集中阅读候选观察池。
- 如果候选已经被多个周期反复验证，且有真实行动反馈，可以提醒用户考虑更新 `01_core/道/`、`01_core/法/` 或相关 `术/`。
- 如果候选仍未验证，保留在候选观察池，继续观察。
- 如果候选被反例推翻或明显过期，建议用户删除或标记为放弃。
- Codex 不能自动把候选观察升级为正式 `道/`、`法/`、`术`；必须先报告证据和建议，由用户确认。

## 无损整理口径

可以做：

- 去掉 `- [x]` / `* [x]` 标记。
- 保留原句主体和标点。
- 保留 `Memory：`、`继续追踪：`、`道：`、`法：`、`术：` 等来源标签。
- 完全重复项只保留一次。
- 如果原句很长，可以换行排版，但不能删掉关键语义。

不可以做：

- 把用户勾选的完整判断改写成更短的新句。
- 删除限定语、反例、对象、时间范围或比喻。
- 因为数量多而丢弃已勾选项。
- 把未勾选项当成已确认 Memory。
- 把候选观察直接升级为正式 `道/`、`法/`、`术`。

## 候选不足

如果没有已勾选内容，也没有用户明确确认的内容，Codex 应报告“本期不建议写入 Memory”，不要硬凑。
