# Memory 输出规则

Memory 是人工确认后的跨期上下文，用于承接已确认内容、继续追踪和候选观察。它不是 Output 摘要，也不是正式 `道/`、`法/`、`术` 入库结论。

## 核心原则

人工确认内容优先无损迁移。

* 已勾选 checkbox 或用户明确要求保留：必须进入 Memory，除非用户撤销。
* 不设数量上限；用户确认多少，就迁移多少。
* 不做有损压缩，不改写关键限定、比喻、因果、对象和判断。
* 未勾选内容默认不写入；只能作为候选观察或继续追踪。
* 道 / 法 / 术候选只进入候选观察池，不能自动升级为正式文件。
* 没有确认内容时，报告“本期不建议写入 Memory”，不要硬凑。

## 输入

主要读取：

* `04_output/weekly/YYYY-WW.md`
* `04_output/monthly/YYYY-MM.md`
* `04_output/yearly/YYYY.md`
* 对应 `_dist/**/memory-candidates.md`

重点看：

* 已勾选 checkbox：`- [x]`、`* [x]`
* 明确标记为「进入记忆」「继续追踪」「重要」「保留」「确认」的内容
* 用户当场明确要求保留的内容
* 明确标记为「芒格之魂手动洞察」的内容

## 输出位置

* Weekly / Monthly 写入：`01_core/memory/YYYY-QN.memory.md`
* Yearly 写入：`01_core/memory/YYYY.memory.md`

## 推荐结构

候选观察统一放在季度 / 年度 Memory 文件顶部：

```md
## 候选观察池

### 道候选
### 法候选
### 术候选
### 芒格之魂手动洞察候选

## YYYY-Www

### Memory
### 继续追踪

## Monthly｜YYYY-MM

### Memory
### 继续追踪

## Yearly｜YYYY

### Memory
### 继续追踪
```

具体周期小节只保留 `Memory` 和 `继续追踪`。
`值得作为道 / 法 / 术候选观察` 统一迁移到顶部候选观察池，并保留来源标签。

## 候选观察池维护

写入规则：

* 已勾选的道 / 法 / 术候选观察必须进入候选观察池。
* 条目前必须保留来源，例如 `[2026-W22]`、`[Monthly｜2026-05]`。
* 原文无损迁移，不改写候选判断。
* 同一候选重复出现时，可以追加来源；有语义差异时不要强行合并。

季度审查：

* 多周期反复验证、且有真实行动反馈的候选，可以提醒用户考虑更新 `01_core/道/`、`01_core/法/` 或相关 `术/`。
* 未验证的候选继续观察。
* 被反例推翻或明显过期的候选，建议删除或标记放弃。
* 执行者不能自动升级候选；必须先报告证据和建议，由用户确认。

## 芒格之魂手动洞察候选

芒格之魂手动洞察是候选观察池中的独立候选类型。

写入规则：

* 用户明确标记或勾选的「芒格之魂手动洞察」，必须全量进入 `### 芒格之魂手动洞察候选`。
* 保留原文锋芒、反转、隐喻和判断力度，不压缩成普通摘要。
* 可以统一 bullet、来源标签和排版。
* 默认只是候选，不自动进入正式 `道/`、`法/`、`术`。
* 后续是否升级，必须由用户人工确认。

格式：

```md
- [2026-W23｜主题] 洞察：……
```

## 无损整理口径

可以做：

* 去掉 `- [x]` / `* [x]` 标记。
* 统一 bullet、来源标签和小节。
* 完全重复项只保留一次。
* 长句可换行排版，但不能删掉关键语义。

不可以做：

* 改写用户已确认的完整判断。
* 删除限定语、反例、对象、时间范围或比喻。
* 因数量多而丢弃已确认项。
* 把未勾选项当成已确认 Memory。
* 把候选观察直接升级为正式 `道/`、`法/`、`术`。
* 把芒格之魂手动洞察压缩成普通摘要。

## 候选不足

如果没有已勾选内容，也没有用户明确确认的内容，输出：

> 本期不建议写入 Memory：没有已勾选或明确确认的内容。
