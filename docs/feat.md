# 功能与 UI 标准 (FEAT)

## 1. 学习引擎 (Learning Engine)
系统需支持顶级产品模块切换。顶级菜单不放在阅读侧边栏中，避免未来模块增长后不可维护。

- **浏览**：阅读整个仓库，包含搜索、目录、文件内容和双链预览。
- **学习**：独立的学习动力引擎，切换后不显示浏览模式的搜索、目录或文件正文。
- **行动**：预留给未来“行动动力引擎”，承接知行合一、实验、复盘和真实世界反馈。
- 顶级菜单由 `code/public/config.js` 的 `APP_CONFIG.menu` 配置生成。

- Chat Pack 的上下文必须动态递归扫描当前仓库文件系统。
- 忽略 `.git`、`node_modules`、`dist`、`build`、`.next`、`.vite`、`code/`、`AGENTS.md` 和 `CONTEXT_MASTER.md`。

### A. 浏览模式 (Browse)
- 左侧保留仓库 Markdown 文件树与搜索。
- 支持 `[[双链]]` 与内部链接的点击/预览。

### B. 学习模式 (Learning)
- **Chat Pack 生成器 P0**：支持输入当前问题、选择 Prompt、勾选 Context 来源、预览完整 Chat Pack，并通过一个主按钮生成且复制到任意 AI。
- 生成 Chat Pack 时，应基于当前勾选来源即时生成 Context 后再组装。
- **全局认知审计**：递归打包全库核心 Markdown，驱动 AI 进行逻辑漏洞、道法脱节和异化风险扫描。
- **领域研究**：默认打包 `README.md + 法/<当前领域>/`，围绕用户自定义问题研究领域结构、边界、反例和行动抓手。
- 领域必须从 `法/<领域>/` 自动发现，不能写死为教育、投资、ai。
- 每个场景需要独立 Prompt 模板，且支持按场景和领域在浏览器本机保存编辑后的版本。
- Prompt 正文由仓库根目录的 `meta-prompts/<id>.md` 维护，学习场景 `id` 同时作为 Prompt 文件 ID。
- Prompt 目录由 `code/public/config.js` 的 `APP_CONFIG.promptDirectory` 配置。
- 生成前允许勾选上下文来源，支持场景推荐路径一键恢复。
- 学习面板需要展示文件数、字节大小和进度条。
- 学习面板需要保留更新日志区域，用于记录 AI 回答后的新洞察。

## 2. UI 体验标准
- **风格**：极简、冷色调、留白充足、美观、可以参考飞书和飞书文档。
- **交互**：操作必须直接（阅读、搜索、一键封装上下文）。
- **布局**：适配长文档，优先考虑文本的可读性。
