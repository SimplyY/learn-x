# Learn-X

> Learn-X 是我的个人认知进化系统：以「道」校准方向，以「法」形成判断，以「术」推动行动，以「器」降低成本，让 AI 成为我的剑，而不是我的茧。

## 1. 核心目标

当前 P0：

1. 维护我的「道」与「法」。
2. 一键生成可复制给任意 AI 的 Chat Pack。
3. 用 AI 辅助学习、复盘、自我认知和真实行动。

核心公式：

```text
AI 效果 = Prompt × Context × Model × Feedback × Action
```

当前系统重点自动化：

```text
Prompt + Context → Chat Pack
```

每周学习闭环：

```text
信息输入 + 行动证据 + 自我反馈
↓
Weekly Output
↓
人工判断
↓
Memory
↓
下一轮行动
```

## 2. 核心思想

* **为学日增**：flomo、飞书、书籍、播客、AI 对话、真实项目，用于积累术与器。
* **为道日减**：Git 仓库沉淀、压缩、删减真正重要的道与法。
* **求真**：对自己、知识、他人、现实反馈真诚。
* **知行合一**：没有行动验证的洞察，不能轻易进入道和法。

## 3. 道法术器

| 层级 | 含义 | 作用 |
| -- | -- | -- |
| 道 | why，价值层 | 方向、本质、人生母题 |
| 法 | what，认知框架层 | 教育观、投资观、AI 观、创业观 |
| 术 | how，方法执行层 | 方案、流程、Prompt、复盘模板 |
| 器 | tool，工具实现层 | flomo、飞书、Codex、脚本、书籍 |

原则：

```text
道要少，法要精。

道：我为何而活。
法：我如何判断。
术：我如何行动。
器：我用什么实现。
```



## 4. Chat Pack

高频使用场景：

```text
当前问题 + 已选 Prompt + 已选 Context + 输出要求 = Chat Pack
```

目标：

1. 电脑端 GUI 勾选上下文和 Prompt。
2. 手机端快速选择、一键复制。
3. 可复制到 ChatGPT、Claude、Gemini、豆包、Kimi 等 AI。
4. 飞书 Prompt 合集作为编辑源，Learn-X 作为调用与组装入口。

核心规则：

```text
道是默认上下文。
法是主题上下文。
术是任务上下文。
器是执行上下文。
Prompt 是调用方式。
Chat Pack 是交付形态。
```



## 7. 学习模型

Learn-X 长期借鉴：

* 苏格拉底：对知识诚实，寻找反例，检验可证伪性。
* 费曼：输出倒逼输入。
* 芒格：多元思维模型，反过来想。
* 马斯克：第一性原理。
* 维特根斯坦：精确语言即认知进化。
* 贝叶斯：根据证据更新判断。
* 演化生物学：系统在选择和反馈中进化。
* 控制论：没有反馈，就没有控制。

## 11. Input 边界

```text
03_input/inbox  记录我接收了什么
03_input/action 记录我做了什么
03_input/log    记录我如何理解自己
03_input/index  记录本次处理什么
```

`03_input/index/weekly/*.index.md` 是处理清单；`03_input/log/weekly/*.review.md` 才是真正周记。`01_core/memory/` 是人工确认后的输出，不和 log 混淆。
