   - 通知从“每天都提醒”改成“只在策略触发时提醒”。
   - 这种改动看起来小，但直接改善了信号质量，减少了不必要的打扰。

## 核心项目与成果

### Learn-X

- 关键成果：把输入 / 输出 / Memory / Process Pack 的边界继续钉死，减少脚本越权和人工误用。
- 关键成果：本地 Chat Pack 侧更像一个可搜索、可编辑、可验证的知识工作台，而不是一坨静态 Prompt。
- 关键成果：`app/code/`、`docs/TECH.md` 和 `.agents/skills/learn-x-process/` 的职责划分更清晰，后续维护成本更低。
- 重要性：这是整个个人认知系统的底座，其他项目的输出最终都要回到这里。

### Health-X

- 关键成果：健康周报已经能同步回 Learn-X 的周输入目录。
- 关键成果：评分、字段校验、洞察输出和飞书适配层都有加固，说明同步链路不是一次性脚本。
- 重要性：把健康数据从“外部报表”变成“可回流证据”，更符合 Learn-X 的闭环原则。

### invest-update-yzyx

- 关键成果：月度资产更新流程被重组为更清晰的技能与配置体系。
- 关键成果：输入、审计、输出、快照、操作记录的分工更明确，后面更容易做日级审计。
- 重要性：这个仓库的价值在于高频、确定性、可追溯，重构后更接近这个目标。

### Research-X

- 关键成果：研究提示词、工作文档模板和版本入口被重新整理。
- 重要性：研究仓不是收集仓，整理提示词资产能减少每次启动研究时的上下文摩擦。

### Invest

- 关键成果：通知策略收紧，避免把日常噪声当成有效信号。
- 重要性：投资相关系统最怕提醒泛滥，降噪本身就是提高决策质量。

## 跨项目能力变化

- 从“单仓做功能”转成“多仓共享边界感”：Learn-X、Health-X、Research-X、资产更新仓都在强调输入、处理中间态、最终输出的分层。
- 从“结果导向”转成“证据导向”：越来越多的变化不是多出一个页面，而是多出一个可验证、可回溯的中间层。
- 从“手工搬运”转成“确定性写回”：Health-X 写回 Learn-X，Learn-X 生成 Process Pack，资产更新仓按日留痕，都是同一类能力。
- 从“工具堆叠”转成“提示词和规则资产化”：Research-X 和 Learn-X 的提示词整理说明，Prompt 本身已经变成要维护的资产。

## 对 AI Builder / 学习者 / 知行合一的意义

- 对 AI Builder：这周最有价值的不是又做了一个界面，而是把 Prompt、Context、校验、写回变成了可以版本化的契约。
- 对学习者：周复盘不再只是“看到了什么”，而是开始明确“哪些证据值得留下、哪些结论只能短期存在”。
- 对知行合一：Health-X 和 Invest 都在把行动结果变成可追踪证据，减少“知道了但没落地”的空转。

## 必须做 / 值得做 / 暂时不做

- 必须做：继续把 Learn-X 的周输入、Health-X 回流、资产更新和研究提示词维护成可重复的证据链。
- 必须做：把仍在漂移的本地工具和脚本补上最小测试，避免流程边界再次被改坏。
- 值得做：把跨仓库的复盘摘要再压缩成一页式总览，减少人工串联成本。
- 暂时不做：不要把这些项目再抽一层“大一统工作流”框架，当前问题不是框架不够大，而是边界还不够硬。

## 当前最缺

- 最缺的是一条稳定的跨项目周级总览能力，能把“有哪些高价值变更”自动压缩成可读摘要。
- 次缺的是更少的人肉搬运，尤其是从 Git diff、周输入和输出正文之间的对照。
- 还缺的是统一的质量门禁表达，让不同仓库的“写回、同步、审计、测试”口径更一致。
