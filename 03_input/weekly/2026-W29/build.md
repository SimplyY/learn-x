### 最核心的 5 件事

#### 1. Skill 治理由“整理资产”升级为“可验证控制面”

- `skills` 建立健康评分、群规则证据采集、触发词规范化和全局治理入口。
- 评分只扣已确认缺陷，避免把“未检查”误判为“有问题”。
- `neat-all`、`diffall`、`thinking-group` 开始形成相互校验的治理链。
- 价值：跨项目规则不再只靠记忆，能被检查、解释和持续修正。

闭环状态：主体已闭环；仍需观察规则是否产生误报和治理噪声。

#### 2. 飞书 Codex 入口从“能跑”推进到“可长期运行”

- `lark-channel-bridge` 修复 daemon 状态、运行时迁移、模型配置和重复终稿。
- 图片处理回到 bridge 内 OCR 隔离路径，普通 Codex 线程不再直接承接图片输入。
- Time-X 三图仅在精确入口、三张图片、同一发送者时释放路径。
- 回复结构、引用和媒体并行处理进一步降低移动端使用摩擦。

闭环状态：主要路径已有测试；真实飞书长周期稳定性仍需继续观察。

#### 3. Time-X 同时打通“随时记”和“每周屏幕时间”

- 日历事项支持六类交叉标签、幂等键、主日历约束和写后回读。
- `time-x-note` 增加本地原文存储、系统锁和原子替换，避免写坏月文件。
- 屏幕时间转为 Learn-X Stage 1.5，由桌面 Codex 目视三图，只传结构化结果。
- Android 与 Mac 采用各自正确的时间键，并以双设备成功作为 Stage 2 门禁。

闭环状态：实现和测试已完成；仍缺一次完整真实周的双设备非 dry-run 验收。

#### 4. Learn-X 的周输入证据更完整，也更尊重事实边界

- 新增日历计划输入，只保存聚合时长、实例数和可用状态，不复制日程正文。
- 修正上海时区全天日程解析，并用测试锁定聚合与失败覆盖行为。
- 周输入可并行准备，但屏幕时间双设备结果仍是进入 Stage 2 的硬门禁。
- 上线无上下文公共 Chat Pack，继续收紧本地私有输入与公共构建的边界。

闭环状态：代码边界已闭环；本周完整输入到 Output 的人工链路尚未在此证据中确认。

#### 5. Research-X 从本地素材库转为 Skill + 飞书数据源

- 新增“大佬情报”检索，支持实时人物/来源查询和逐人完成门禁。
- Prompt、脚本和测试下沉到各自 Skill，历史素材模板从仓库移除。
- 生产人物与来源数据以飞书 Base 为唯一真值，不建立本地镜像。
- 价值：研究入口更轻，数据职责更清晰，也减少历史材料拖累上下文。

闭环状态：结构迁移已完成；当前工作区仍有相关 Skill 与 README 未提交改动。

### 核心项目与成果

- `skills`：把 Skill/Group 管理变成有证据、有评分、有硬门禁的共享控制面。
- `lark-channel-bridge`：稳住移动入口、运行时、模型和图片安全边界。
- `time-x`：形成日历随时记与屏幕时间两条可验证的个人行动证据链。
- `learn-x`：接收日历计划和屏幕时间证据，同时保持隐私与事实语义。
- `research-x`：把研究执行收敛到 Skill，把生产数据收敛到飞书 Base。
- `read-x`：公共长文发布改为匿名访问验证失败即停止，避免错误暴露链接。
- `invest-x` / `invest-log`：投资速查、卡片输出和温度提醒入口继续收敛。

### 对 AI Builder、学习者与知行合一的意义

- 对学习者：日历计划、屏幕时间、阅读和研究可成为互相校验的周证据。
- 对知行合一：记录不再只是叙述，开始带有时间、结果、失败和回读状态。
- 风险也更清楚：流程更强不代表行动已发生，计划数据不能冒充完成证据。
